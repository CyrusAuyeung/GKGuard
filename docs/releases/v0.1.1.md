# GKGuard v0.1.1

## 中文

## 重点更新

- 将 C2 后端打包进桌面 Release，减少用户手动启动后端的要求。
- 桌面应用启动时会尝试拉起本地后端，再加载 C2 演示界面。
- 延续 v0.1.0 的 C2 mock dashboard、报告、审计和导出能力。
- 继续验证 GitHub Actions 到 GitHub Release 的桌面发版链路。

## 安装与网络说明

- 本版本开始向“安装后打开桌面端即可进入 C2 演示”靠拢。
- C2 mock 演示不依赖校园网；真实 C1、校园视频或服务器检索能力在该版本尚未接入。
- 该版本资产文件名仍保留早期点号格式，并且安装包名显示为 `0.1.0`，这是历史打包命名问题；建议新试用优先下载最新 Release。

## 验证情况

- GitHub Actions 完成后端打包、Electron 构建和 Release 资产上传。
- 已验证桌面端可随包携带本地后端资源。

## 安装包

本 Release 下方包含：

- `GKGuard.Setup.0.1.0.exe`：v0.1.1 发布中的 Windows 桌面安装包，文件名为历史格式。
- `GKGuard.Setup.0.1.0.exe.blockmap`：Electron 更新差分元数据，文件名为历史格式。
- `latest.yml`：Electron 更新通道元数据。

普通试用建议下载 `.exe` 文件；`.blockmap` 和 `latest.yml` 通常不需要手动打开。

# GKGuard v0.1.1

## English

## Highlights

- Bundled the C2 backend into the desktop Release, reducing the need to start the backend manually.
- The desktop app now attempts to start a local backend before loading the C2 demo UI.
- Continued the v0.1.0 C2 mock dashboard, report, audit, and export capabilities.
- Continued validating the GitHub Actions to GitHub Release desktop publishing flow.

## Installation And Network Notes

- This version moves toward an install-and-open C2 demo experience.
- The C2 mock demo does not require campus network access; real C1, campus video, or server-side search was not integrated in this version.
- Asset names still use the early dotted naming format, and the installer name shows `0.1.0`; this is a historical packaging-name issue. Prefer the latest Release for new testing.

## Verification

- GitHub Actions completed backend bundling, Electron build, and Release asset upload.
- Verified that the desktop package can carry local backend resources.

## Installer Assets

This Release includes:

- `GKGuard.Setup.0.1.0.exe`: Windows desktop installer for v0.1.1, using the historical filename format.
- `GKGuard.Setup.0.1.0.exe.blockmap`: Electron differential update metadata, using the historical filename format.
- `latest.yml`: Electron update-channel metadata.

For normal testing, download the `.exe` file; `.blockmap` and `latest.yml` usually do not need to be opened manually.
