# GKGuard v0.1.29

## 重点更新

- 修复上传区交互：上传图片后，点击上传区图片或空白区域会重新打开文件选择器；“放大选择目标人脸”改为显式按钮，不再占用上传区点击行为。
- 修复结果页人物照片来源：人物照片优先使用 CampusVision C1 返回的人脸缩略图，只有缺失时才回退到前端本地裁切，避免左侧人物照片与检索记录缩略图不一致。
- 修复前端本地裁切 fallback：当 CampusVision C1 只返回归一化 bbox 时，不再把 `0.x` 坐标误当作像素坐标。
- 修复多人选择弹窗缩放：弹窗默认按容器完整适配原图，`缩小` 可继续低于适应比例，`适应` 回到完整显示，`放大` 继续用于精细点选。
- 同步 README、API 规范、CampusVision C1 / GKGuard C2 集成说明、自动连接说明、演示脚本、文档索引、图标说明和版本号到 `v0.1.29`。

## 行为说明

- 多人上传图仍会自动提示选择目标人脸；关闭弹窗后，可通过显式按钮重新打开大图选择，也可直接点击上传区重新上传新图片。
- 搜索结果左侧人物照片与检索记录缩略图使用同一优先来源：CampusVision C1 返回的人脸缩略图优先，前端查询图裁切只作为缺失兜底。
- 本版本不向仓库加入真实监控、人脸或个人信息数据；本地测试图片继续位于被 git 忽略的 `local-test-data/`。

## 验证

- 通过：`node --check backend/app/static/app.js`。
- 通过：`python -m pytest backend`。
- 通过：`npm run test:e2e`，覆盖上传区重新选择文件、显式放大选人按钮、多人弹窗适应/缩小、结果页人物照片使用 CampusVision C1 缩略图、关键帧目标框、无匹配/超时恢复和响应式布局。
- 通过：in-app Browser 验证 `/demo?desktop=1&asset=v0.1.29-ui` 页面身份、非空渲染、无 console warning/error，并确认无上传图时本地模拟结果页可响应进入；上传区按钮、弹窗缩放和结果页人物照片来源由 Playwright E2E 覆盖。
- 发布流程要求 GitHub Actions `Verify` 和 Release workflow 通过，并在正式 GitHub Release 中核对安装包、`.blockmap` 和 `latest.yml` 齐全。

## 安装

普通首次安装下载并运行 `GKGuard-Setup-0.1.29.exe`。已安装旧版时，可使用软件右上角 `检查更新` 入口更新。

真实 CampusVision C1 检索仍需要校园网、VPN 或 SSH 隧道能访问 CampusVision C1 服务；未上传图片时可使用本地模拟流程演示，上传图片后的检测失败、无匹配、超时或真实检索失败不会展示模拟命中。

# GKGuard v0.1.29

## Highlights

- Fixed upload-area interaction: after an image is uploaded, clicking the uploaded image or empty upload area opens the file picker again. Enlarged target-face selection is now an explicit button instead of taking over the upload-area click.
- Fixed result portrait source priority: the result portrait now prefers the CampusVision C1 face thumbnail and falls back to frontend local cropping only when that thumbnail is missing, avoiding mismatches with record thumbnails.
- Fixed frontend local-crop fallback: normalized `0.x` bounding boxes from CampusVision C1 are no longer treated as pixel coordinates.
- Fixed multi-face modal zooming: the modal fits the full original image by default, `缩小` can go below the fit scale, `适应` returns to full-image fit, and `放大` remains available for precise target selection.
- Synchronized README, API specification, CampusVision C1 / GKGuard C2 integration notes, auto-connection guide, demo script, documentation index, icon notes, and package versions to `v0.1.29`.

## Behavior Notes

- Multi-face uploads still prompt the user to select a target face. After closing the modal, users can reopen it through the explicit button or click the upload area to upload a new image.
- The result-side portrait and record thumbnails now share the same source priority: CampusVision C1 face thumbnails first, frontend query-image crops only as a fallback.
- This release does not add real surveillance, face, or personal-information data to the repository; local test images remain under git-ignored `local-test-data/`.

## Verification

- Passed: `node --check backend/app/static/app.js`.
- Passed: `python -m pytest backend`.
- Passed: `npm run test:e2e`, covering upload-area re-selection, explicit enlarged selection button, multi-face modal fit/zoom-out behavior, CampusVision C1 thumbnail result portrait, keyframe target overlays, no-match/timeout recovery, and responsive layout.
- Passed: in-app Browser validation at `/demo?desktop=1&asset=v0.1.29-ui` for page identity, non-empty rendering, zero console warnings/errors, and responsive entry into the mock results screen without an uploaded image; the upload-area button, modal zooming, and result portrait source are covered by Playwright E2E.
- Release flow requires GitHub Actions `Verify` and the Release workflow to pass, followed by checking that the official GitHub Release contains the installer, `.blockmap`, and `latest.yml`.

## Installation

For a first-time installation, download and run `GKGuard-Setup-0.1.29.exe`. Existing installations can update through the top-right `检查更新` entry.

Real CampusVision C1 search still requires campus network, VPN, or an SSH tunnel to the CampusVision C1 service. The mock flow remains available when no image has been uploaded; uploaded-image detection failures, no-match results, timeouts, or real-search failures do not show mock hits.
