# GKGuard v0.3.0

`v0.3.0` 是一次 GKGuard C2 前端信息架构与桌面工作台体验更新。版本保留既有 CampusVision C1 真实检索、查询图目标人脸选择、人物特征检索、结果关键帧标注、路线联动、媒体缓存隔离、受信 SSH 主机密钥自动比对、跨平台桌面打包和本地 mock 回退能力，同时把前端重构为更清晰的工作台式布局。

## 变更摘要

- 重构 GKGuard C2 静态前端为固定工作区导航：桌面端左侧导航提供 `人脸以图搜人` 与 `人物特征搜索` 两个入口，中等宽度和移动视口自动收敛为紧凑布局。
- 重构结果页为证据优先布局：左侧保留目标人物与检索记录，右侧展示关键帧、相似度、属性摘要、时间线和操作按钮。
- 新增候选人物抽屉与事件详情抽屉，用于在不离开当前结果上下文的情况下复核候选人物、事件记录、相机、时间、匹配类型和属性。
- 调整响应式布局，避免中等宽度下人物照片、数据来源、命中记录、操作按钮和结果记录互相遮挡。
- 将 Impeccable 设计约束引入仓库，新增 `PRODUCT.md`、`DESIGN.md`、`.github/skills/impeccable/` 和相关配置，明确 GKGuard C2 Workbench / Evidence Desk 的视觉方向。
- 同步 README、文档索引、API 规范、数据字典、CampusVision C1 / GKGuard C2 集成说明、自动连接说明、演示脚本、支持与安全文档、Issue 模板和 AI agent / 协作流程规范。

## 使用影响

- Windows 客户端仍可通过右上角 `检查更新` 获取新版本；Windows Release 继续提供 `GKGuard-Setup-0.3.0.exe`、`.blockmap` 和 `latest.yml`。
- macOS 和 Linux 仍通过 GitHub Release 下载对应平台安装文件；macOS 包当前仍是未签名/未公证的内部测试包。
- CampusVision C1 代理接口保持兼容。本版本主要改变 GKGuard C2 前端展示结构和文档/设计约束，不要求 CampusVision C1 服务端为本次 UI 更新额外变更。
- 本地 mock 回退仍可在离线或非校园网环境演示工作台流程；上传真实查询图后的真实检索仍要求可访问 CampusVision C1。

## 验证

本版本准备过程中执行并通过：

- `node --check backend/app/static/app.js`
- `node --check desktop/main.js`
- `node --check desktop/preload.js`
- `node --check tests/e2e/gkguard-ui.spec.js`
- `python -m pytest backend`
- `python -m py_compile backend/desktop_server.py backend/app/services/c1_service.py`
- `npm run test:e2e`
- `npm audit --audit-level=low`
- `git diff --check`

浏览器人工验证覆盖桌面 `1280x720`、中等宽度 `820x920`、移动视口 `390x720`、人脸以图搜人流程、人物特征搜索入口、候选人物抽屉、事件详情抽屉、路线页和结果记录切换。

## 安全与数据边界

- 本版本不提交真实监控视频、真实查询图片、抽帧图、人脸裁剪图或真实人员身份数据。
- 文档示例继续使用占位数据，不公开真实运行环境凭据、SSH 私钥、token、API key、`known_hosts` 行或 `authorized_keys` 行。
- 模型相似度和检测置信度仍作为辅助判断展示，不表示绝对身份确认。

# GKGuard v0.3.0

`v0.3.0` is a GKGuard C2 frontend information-architecture and desktop workbench update. It keeps the existing CampusVision C1 real-search integration, query-face target selection, person-attribute search, keyframe target overlays, route linkage, media-cache isolation, trusted SSH host-key comparison, cross-platform desktop packaging, and local mock fallback, while restructuring the frontend into a clearer workbench layout.

## Summary

- Reworked the GKGuard C2 static frontend around fixed workspace navigation: desktop layouts expose `人脸以图搜人` and `人物特征搜索` in the left navigation, while medium-width and mobile viewports collapse into compact layouts.
- Reworked the result page into an evidence-first layout: the left side keeps the target person and result records, and the right side presents keyframes, similarity, attribute summaries, the timeline, and actions.
- Added candidate-person and event-detail drawers so operators can inspect candidate people, event records, cameras, times, match types, and attributes without leaving the current result context.
- Adjusted responsive layout so medium-width views do not let the person portrait, source summary, hit count, action buttons, or result records overlap.
- Added the Impeccable design constraints to the repository through `PRODUCT.md`, `DESIGN.md`, `.github/skills/impeccable/`, and related configuration, defining the GKGuard C2 Workbench / Evidence Desk visual direction.
- Synchronized README, documentation index, API specification, data dictionary, CampusVision C1 / GKGuard C2 integration guide, auto-connection guide, demo script, support and security docs, Issue templates, and AI-agent / collaboration workflow rules.

## User Impact

- Windows clients can still use the top-right `检查更新` entry to get the new version; the Windows Release continues to provide `GKGuard-Setup-0.3.0.exe`, `.blockmap`, and `latest.yml`.
- macOS and Linux still download platform-specific packages from GitHub Releases; the current macOS package remains an unsigned and unnotarized internal test build.
- CampusVision C1 proxy APIs remain compatible. This release mainly changes the GKGuard C2 frontend presentation structure and documentation/design constraints, and does not require an extra CampusVision C1 server change for the UI update itself.
- Local mock fallback can still demonstrate the workbench flow offline or outside the campus network; real uploaded-query-image search still requires CampusVision C1 access.

## Validation

This release preparation passed:

- `node --check backend/app/static/app.js`
- `node --check desktop/main.js`
- `node --check desktop/preload.js`
- `node --check tests/e2e/gkguard-ui.spec.js`
- `python -m pytest backend`
- `python -m py_compile backend/desktop_server.py backend/app/services/c1_service.py`
- `npm run test:e2e`
- `npm audit --audit-level=low`
- `git diff --check`

Manual browser verification covered desktop `1280x720`, medium-width `820x920`, mobile `390x720`, the face-image search flow, the person-attribute search entry, candidate-person drawer, event-detail drawer, route page, and result-record switching.

## Security And Data Boundary

- This release does not commit real surveillance videos, real query images, extracted frames, face crops, or real person-identity data.
- Documentation examples continue to use placeholders and do not disclose real runtime credentials, SSH private keys, tokens, API keys, `known_hosts` lines, or `authorized_keys` lines.
- Model similarity and detection confidence remain assistive signals and do not represent absolute identity confirmation.
