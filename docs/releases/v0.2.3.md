# GKGuard v0.2.3

## 变更摘要

`v0.2.3` 是 `v0.2.2` 之后的安全修复与流程文档补丁版本。本版不新增主要产品入口，重点修复 CampusVision C1 视频重建索引失败时可能破坏既有索引状态的问题，并同步修正协作模板说明，确保后续 PR 正文和 review 流程表述保持一致。

## 主要更新

- CampusVision C1 视频重建索引流程在写入新索引前先进入暂存阶段，不再把原始帧数组长期保留在采样列表中，降低失败路径上的内存占用和状态污染风险。
- CampusVision C1 重建索引在提交阶段失败时会回滚数据库记录和帧目录；若该视频此前已有可用索引，失败后会恢复旧事件、人物观测、人脸记录、人物索引状态和相关 appearance sessions，而不是留下半写入的新索引或删除旧索引。
- CampusVision C1 数据库写操作增加写锁保护，避免重建索引期间与其他写入流程交错；只读接口仍保持普通连接路径，不因重建索引锁产生全局串行化。
- CampusVision C1 查询图超限路径继续清理临时上传目录，并返回 413，避免异常输入留下孤立临时文件。
- 贡献文档中的 PR 模板说明保持与 `.github/PULL_REQUEST_TEMPLATE.md` 一致，Issue 模板默认版本同步为 `v0.2.3`，避免协作者误解 PR 正文格式、检查项或问题反馈版本。
- README、文档索引、SECURITY、SUPPORT、API 规范、CampusVision C1 / GKGuard C2 集成说明、自动连接说明、演示脚本、CampusVision C1 文档和任务记录同步为 `v0.2.3` 当前状态。

## 行为边界

- 本版延续 `v0.2.0` 到 `v0.2.2` 的产品能力边界：人物特征检索仍是可解释筛选与相似事件检索，不是身份确认。
- 重建索引失败后的恢复只保证 CampusVision C1 已提交的旧索引状态不被失败任务破坏；如果首次索引失败且此前没有旧索引，视频仍会进入失败状态并需要重新索引。
- 查询图不可解码、超限或人物特征条件校验失败属于输入校验失败，不表示 CampusVision C1 服务不可用；只有连接失败、健康检查失败或脱敏后的服务端 5xx 才进入连接恢复或重试路径。
- 本仓库不提交真实视频、真实查询图片、抽帧图、人脸裁剪图、人体裁剪图、数据库、`.env`、服务器凭据或个人信息。

# GKGuard v0.2.3

## Summary

`v0.2.3` is a security-fix and workflow-documentation patch after `v0.2.2`. It does not add major product entries. It focuses on preventing failed CampusVision C1 video re-index jobs from corrupting an existing index state, and aligns collaboration-template wording so future PR bodies and review workflow guidance stay consistent.

## Highlights

- CampusVision C1 video re-indexing now stages sampled frames before committing the new index, and no longer keeps raw frame arrays in the sampled-item list, reducing memory pressure and state pollution on failure paths.
- If CampusVision C1 re-indexing fails during the commit phase, it restores database rows and the previous frame directory. When the video already had a usable index, the old events, person observations, face rows, person-index state, and related appearance sessions are restored instead of leaving a partially written new index or deleting the old one.
- CampusVision C1 database writes now use a write lock so re-indexing does not interleave with other write flows. Read-only APIs continue to use the normal connection path and are not globally serialized by the re-index lock.
- The CampusVision C1 over-limit query-image path continues to clean temporary upload directories and returns 413, preventing invalid input from leaving orphaned temporary files.
- The contribution guide now keeps its PR-template guidance aligned with `.github/PULL_REQUEST_TEMPLATE.md`, and Issue-template default versions now use `v0.2.3`, reducing ambiguity around PR body format, checklist items, and issue-report version context.
- README, docs index, SECURITY, SUPPORT, API specifications, CampusVision C1 / GKGuard C2 integration notes, auto-connection notes, demo script, CampusVision C1 docs, and task records now describe `v0.2.3` as the current state.

## Behavior Boundary

- This release keeps the product boundary from `v0.2.0` through `v0.2.2`: person-attribute search remains explainable filtering and similar-event retrieval, not identity confirmation.
- Recovery after a failed re-index job only guarantees that a previously committed CampusVision C1 index is not damaged by the failed job. If a first-time index has no previous index to restore, the video still enters a failed state and must be indexed again.
- Undecodable or over-limit query images and person-attribute condition validation failures are input-validation failures, not CampusVision C1 service availability failures. Only connection failures, health-check failures, or sanitized server-side 5xx responses enter recovery or retry paths.
- The repository does not commit real videos, real query images, extracted frames, face crops, body crops, databases, `.env`, server credentials, or personal information.

## Install And Update

- Windows: download and run `GKGuard-Setup-0.2.3.exe`. Existing installations can update through the top-right `检查更新` entry.
- macOS: download `GKGuard-macOS-*.dmg` or `GKGuard-macOS-*.zip`. The current package is still unsigned and unnotarized for internal testing.
- Linux: download `GKGuard-Linux-*.AppImage` or `GKGuard-Linux-*.deb`.
- The Release should also include `.blockmap`, `latest.yml`, `latest-mac.yml`, and `latest-linux.yml`.
