# GKGuard v0.1.25

## 重点更新

- 修复多人查询图可能直接进入结果页的问题：已上传图片时，GKGuard C2 必须先完成查询图人脸检测；多张人脸必须由用户在原图上选择目标，检测失败不会展示本地模拟命中结果。
- 修复结果页人物照片：优先展示被选中的查询人脸裁切图，其次才使用 CampusVision C1 代表人脸或完整上传图。
- 修复关键帧目标框兼容性：GKGuard C2 适配器兼容像素、归一化和百分比 bbox 字段，详情关键帧和预览弹窗会继续用 `records[].faceBox` 标出目标人脸并显示相似度。
- 修复查询图人脸框点击命中：人脸框使用稳定的块级边框元素，相似度标签不再拦截指针事件，多人图选择按原图目标框位置可靠触发。
- 恢复桌面端检索记录布局：结果页记录列表回到左侧纵向列表，记录缩略图保持在卡片左侧；移动端继续保留横向滑动列表。
- 收紧最大化窗口下的初始上传页布局，减少下方空白。
- CampusVision C1 InsightFace 默认检测尺寸提升为 `INSIGHTFACE_DET_SIZE=1280`，并补充多人图漏检时的联调检查说明。
- 同步 README、API 规范、CampusVision C1 / GKGuard C2 集成说明、数据字典、演示脚本、CampusVision C1 子服务文档、支持/安全文档、文档索引和版本号到 `v0.1.25`。
- 将静态资源缓存版本更新为 `v0.1.25-ui`，避免桌面壳复用旧后台时加载过期页面资源。

## 行为说明

- 未上传图片时，仍可使用本地模拟数据演示 GKGuard C2 工作台流程。
- 已上传图片时，如果查询图人脸检测失败或真实 CampusVision C1 检索失败，页面停留在上传页并提示重试，不进入模拟结果页。
- 查询图只有一张人脸时自动检索；查询图有多张人脸时必须选择目标人脸。
- 检测置信度只表示查询图人脸检测质量；结果页相似度仍表示目标人脸与 CampusVision C1 人物库/关键帧的匹配分数。
- 本版本不向仓库加入真实监控、人脸或个人信息数据。

## 验证

- 通过：`python -m pytest backend`，`27 passed`。
- 通过：`python -m py_compile backend/desktop_server.py`。
- 通过：`node --check backend/app/static/app.js`、`node --check desktop/main.js`、`node --check desktop/preload.js`。
- 通过：`npm run test:e2e`，12 个 Chromium 用例覆盖 `1280x720`、`680x640` 和 `390x720` 下的本地模拟、单人自动检索、多人框选、查询图检测失败拦截、桌面左侧记录列表、结果关键帧框选和预览弹窗；额外 Playwright 视觉 QA 覆盖桌面与移动端查询图目标框中心点选、结果关键帧目标框和预览弹窗目标框。
- 通过：`npm audit --audit-level=low`。
- 通过：`npm run dist`，已生成 `GKGuard-Setup-0.1.25.exe`、`GKGuard-Setup-0.1.25.exe.blockmap` 和 `latest.yml`。
- 发布流程要求 GitHub Actions `Verify` 和 Release workflow 通过，并在正式 GitHub Release 中核对安装包、`.blockmap` 和 `latest.yml` 齐全。

## 安装

普通首次安装下载并运行 `GKGuard-Setup-0.1.25.exe`。已安装旧版时，可使用软件右上角 `检查更新` 入口更新。

真实 CampusVision C1 检索仍需要校园网、VPN 或 SSH 隧道能访问 CampusVision C1 服务；未上传图片时可使用本地模拟流程演示，上传图片后的检测或检索失败不会展示模拟命中。

# GKGuard v0.1.25

## Highlights

- Fixed multi-face query images entering results directly: after an image is uploaded, GKGuard C2 must complete query-face detection first; multiple faces require selecting the target on the original image, and detection failures no longer show mock hits.
- Fixed the result portrait: it now prefers the selected query-face crop, then falls back to the CampusVision C1 representative face or the full upload only when needed.
- Improved target-box compatibility on keyframes: the GKGuard C2 adapter accepts pixel, normalized, and percentage bbox fields, while detail keyframes and preview dialogs keep using `records[].faceBox` for the target overlay and similarity label.
- Fixed query-face box hit testing: face boxes now use stable block-level border elements, similarity labels no longer intercept pointer events, and multi-face target selection reliably follows the target-box position on the original image.
- Restored the desktop result-record layout: records return to a left-side vertical list, with each thumbnail on the left side of its card; mobile keeps the horizontal list.
- Tightened the maximized upload-screen layout to reduce bottom whitespace.
- Raised the CampusVision C1 InsightFace default detection size to `INSIGHTFACE_DET_SIZE=1280` and documented the integration check for multi-face under-detection.
- Synchronized README, API specification, CampusVision C1 / GKGuard C2 integration notes, data dictionary, demo script, CampusVision C1 service docs, support/security docs, documentation index, and package versions to `v0.1.25`.
- Updated the static asset cache key to `v0.1.25-ui` so the desktop shell does not reuse stale page resources from an older backend.

## Behavior Notes

- Without an uploaded image, the local mock flow can still demonstrate the GKGuard C2 workbench.
- After an image is uploaded, query-face detection or real CampusVision C1 search failures keep the user on the upload screen with a retry/error message instead of entering mock results.
- Single-face query images auto-search; multi-face query images require selecting the target face.
- Detection confidence describes query-face detection quality only; result similarity still describes the match between the target face and CampusVision C1 person/keyframe records.
- This release does not add real surveillance, face, or personal-information data to the repository.

## Verification

- Passed: `python -m pytest backend`, `27 passed`.
- Passed: `python -m py_compile backend/desktop_server.py`.
- Passed: `node --check backend/app/static/app.js`, `node --check desktop/main.js`, and `node --check desktop/preload.js`.
- Passed: `npm run test:e2e`, with 12 Chromium cases covering local mock flow, single-face auto-search, multi-face selection, query-face detection failure blocking, the desktop left-side record list, result keyframe overlays, and preview dialog behavior at `1280x720`, `680x640`, and `390x720`; additional Playwright visual QA covered desktop and mobile query-face target-box center selection, result keyframe overlays, and preview-dialog overlays.
- Passed: `npm audit --audit-level=low`.
- Passed: `npm run dist`, producing `GKGuard-Setup-0.1.25.exe`, `GKGuard-Setup-0.1.25.exe.blockmap`, and `latest.yml`.
- Release flow requires GitHub Actions `Verify` and the Release workflow to pass, followed by checking that the official GitHub Release contains the installer, `.blockmap`, and `latest.yml`.

## Installation

For a first-time installation, download and run `GKGuard-Setup-0.1.25.exe`. Existing installations can update through the top-right `检查更新` entry.

Real CampusVision C1 search still requires campus network, VPN, or an SSH tunnel to the CampusVision C1 service. The mock flow remains available when no image has been uploaded; uploaded-image detection or search failures do not show mock hits.
