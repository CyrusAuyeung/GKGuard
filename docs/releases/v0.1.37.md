# GKGuard v0.1.37

## 变更摘要

`v0.1.37` 是 `v0.1.36` 安全加固后的体验与性能跟进版本，重点优化结果页左侧检索记录切换时的关键帧加载。GKGuard C2 前端会保留当前关键帧，预加载并解码目标记录关键帧，加载完成后再替换详情图；快速切换记录时会让旧加载任务失效，关键帧加载失败后允许后续点击重新请求。GKGuard C2 后端对成功读取的 CampusVision C1 媒体响应使用按 CampusVision C1 base URL 隔离的进程内短期缓存，并复用短期 CampusVision C1 健康状态探测结果，减少连续切换记录时的重复网络探测和媒体下载。

## 主要更新

- 结果页切换左侧检索记录时，不再先清空右侧关键帧区域；当前关键帧会保持可见，目标关键帧加载完成后再替换。
- 快速切回已显示记录时会取消旧关键帧加载任务，避免旧 preload 完成后覆盖当前记录的标题、详情和人脸框。
- 关键帧预加载失败会清理对应 URL 缓存，后续重新选择同一记录会再次请求媒体资源。
- 关键帧加载期间显示轻量提示，避免 CampusVision C1 媒体响应较慢时出现整块黑屏。
- 结果列表切换只同步激活状态，不再重建整组记录卡片，减少缩略图重载和布局抖动。
- GKGuard C2 `/c1/media/{kind}/{face_id}` 对成功媒体响应增加进程内短期缓存，缓存键包含当前 CampusVision C1 base URL，默认 300 秒、最多 64 项，可通过 `C1_MEDIA_CACHE_TTL_SEC` 和 `C1_MEDIA_CACHE_MAX_ITEMS` 调整。
- GKGuard C2 普通 CampusVision C1 代理请求复用短期健康状态探测结果，默认 15 秒，可通过 `C1_STATUS_CACHE_TTL_SEC` 调整；`GET /c1/status` 仍执行实时探测，探测失败或身份校验失败会清理对应地址的旧健康缓存。
- 静态资源版本参数更新为 `v0.1.37-ui`，避免安装版继续复用旧前端资源。

## 行为边界

- 本版不改变 CampusVision C1 检索算法、相似度计算、结果排序或 API 字段语义。
- 媒体缓存仅保存在 GKGuard C2 进程内，不写入仓库或持久化数据目录；进程重启后缓存清空。
- 媒体缓存只用于成功读取的关键帧和人脸裁剪图，并按 CampusVision C1 base URL 隔离；CampusVision C1 无匹配、超时、检测失败和真实检索失败仍按既有错误恢复流程处理。

## 安装与更新

- Windows：下载并运行 `GKGuard-Setup-0.1.37.exe`。已安装旧版时，可使用软件右上角 `检查更新` 入口更新。
- macOS：下载 `GKGuard-macOS-*.dmg` 或 `GKGuard-macOS-*.zip`。当前仍是未签名、未公证的内部测试包。
- Linux：下载 `GKGuard-Linux-*.AppImage` 或 `GKGuard-Linux-*.deb`。
- Release 中应同时包含 `.blockmap`、`latest.yml`、`latest-mac.yml` 和 `latest-linux.yml`。

# GKGuard v0.1.37

## Summary

`v0.1.37` is a usability and performance follow-up after the `v0.1.36` security hardening. It focuses on keyframe loading when users switch result records from the left list. The GKGuard C2 frontend keeps the current keyframe visible, preloads and decodes the target record keyframe, and replaces the detail image only after loading finishes; fast record switching invalidates stale load tasks, and failed keyframe preloads can be retried on later clicks. The GKGuard C2 backend also uses a short-lived in-process cache for successful CampusVision C1 media responses scoped by CampusVision C1 base URL, and reuses short-lived healthy CampusVision C1 status probes, reducing repeated network probes and media downloads during consecutive record switching.

## Highlights

- Switching result records no longer clears the right-side keyframe area first; the current keyframe remains visible until the target keyframe is ready.
- Switching quickly back to the currently displayed record cancels stale keyframe loads, so an old preload cannot overwrite the current title, details, or face box.
- Failed keyframe preloads evict the failed URL from cache, so selecting the same record later retries the media request.
- A lightweight loading hint appears while the target keyframe is loading, so slow CampusVision C1 media responses do not create a full black detail panel.
- Result-list switching updates only the active state instead of rebuilding all record cards, reducing thumbnail reloads and layout churn.
- GKGuard C2 `/c1/media/{kind}/{face_id}` now uses a short-lived in-process cache for successful media responses with the current CampusVision C1 base URL included in the cache key, defaulting to 300 seconds and 64 items. Tune it with `C1_MEDIA_CACHE_TTL_SEC` and `C1_MEDIA_CACHE_MAX_ITEMS`.
- Normal GKGuard C2 CampusVision C1 proxy requests reuse short-lived healthy status probes, defaulting to 15 seconds through `C1_STATUS_CACHE_TTL_SEC`; `GET /c1/status` still performs a fresh probe and clears stale healthy cache entries when a probe or identity check fails.
- The static asset parameter is updated to `v0.1.37-ui` so installed apps do not keep using stale frontend assets.

## Behavior Boundary

- This release does not change the CampusVision C1 matching algorithm, similarity calculation, result ordering, or API field semantics.
- Media cache is kept only inside the GKGuard C2 process. It is not written to the repository or a persistent data directory, and it is cleared when the process restarts.
- The media cache applies only to successfully fetched keyframes and face crops, and is scoped by CampusVision C1 base URL. CampusVision C1 no-match, timeout, detection-failure, and real-search-failure states continue to use the existing recovery behavior.

## Install And Update

- Windows: download and run `GKGuard-Setup-0.1.37.exe`. Existing installations can update through the top-right `检查更新` entry.
- macOS: download `GKGuard-macOS-*.dmg` or `GKGuard-macOS-*.zip`. The current package is still unsigned and unnotarized for internal testing.
- Linux: download `GKGuard-Linux-*.AppImage` or `GKGuard-Linux-*.deb`.
- The Release should also include `.blockmap`, `latest.yml`, `latest-mac.yml`, and `latest-linux.yml`.
