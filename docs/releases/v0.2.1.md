# GKGuard v0.2.1

## 变更摘要

`v0.2.1` 是 `v0.2.0` CampusVision C1 / GKGuard C2 功能集成后的稳定性补丁版本。本版不改变主要产品入口，重点处理 `v0.2.0` 合并后 review 发现的 CampusVision C1 查询图清理、重复索引幂等性、人物索引与 appearance session 重建、人物特征路线顺序、路线点与结果记录联动和 API 规范示例问题，并递增 GKGuard C2 前端静态资源版本，避免安装版复用旧页面资源。

## 主要更新

- CampusVision C1 `POST /api/v1/query/face-image` 在候选匹配阶段遇到不可解码查询图时返回 400，遇到查询图上传体积、解码像素数或单边尺寸超限时返回 413，并清理本次临时上传目录，避免异常路径残留查询图片。
- CampusVision C1 重新索引同一 `video_id` 前会清理旧事件、人物观测、人脸记录和旧帧目录，并刷新受影响的人物索引、重建相关 appearance sessions，避免重复索引后事件数、观测数、媒体文件累积、人物代表脸指向已删除人脸媒体或剩余事件继续关联已删除 session。
- GKGuard C2 人物特征检索的路线点按事件时间单独排序，并保留 `recordIndex` / `eventId` 映射回按匹配分排序的结果记录；如果默认最高分记录不在可绘制路线点内，结果页会自动选中第一个可绘制路线点对应记录，避免路线起终点倒序、跳跃、路线页无高亮或点击路线点后选中错误记录。
- GKGuard C2 路线联动兼容未提供 `recordIndex` / `eventId` 的 CampusVision C1 响应；未映射路线点按路线下标维护当前高亮，不再把第 6 个及之后的路线点强行归并到最后一条结果记录。
- API 规范更正 `/c1/query/face-image` 的参数位置：`top_k`、`min_score`、`max_gap_sec`、候选开关、事件/匹配数量和时间筛选均为 Query 参数，`file` 保持 multipart 字段。
- API 规范更正人物特征查询示例：`time_range` 使用 `start_time` / `end_time`，与 CampusVision C1 实际请求模型一致。
- GKGuard C2 静态资源版本参数更新为 `v0.2.1-ui`，桌面端启动时继续清理 renderer cache，并加载新版 CSS/JS。
- 协作流程文档明确 PR 正文 reaction 合并门槛：只有在没有 PR 正文审查信号且需要主动启动审查时，才可评论 `@codex review`；PR 正文已有 👀 或等价“正在 review”信号时，不得再次手动触发 review，只应继续轮询等待结果。
- README、文档索引、SECURITY、SUPPORT、CampusVision C1 / GKGuard C2 集成说明、自动连接说明、演示脚本和图标说明同步为 `v0.2.1` 当前状态。

## 行为边界

- 本版延续 `v0.2.0` 的功能边界：人物特征检索是可解释筛选与相似事件检索，不是身份确认。
- 重复索引同一视频会按该 `video_id` 替换旧索引产物，刷新或删除不再有有效人脸记录的人物条目，并重建受影响人物的 appearance sessions；如索引失败，应重新运行索引，不应把失败状态描述为保留旧事件结果。
- 上装颜色、眼镜状态和外观倾向仍来自 CampusVision C1 模型/profile 输出，不能写成绝对事实。
- 本仓库不提交真实视频、真实查询图片、抽帧图、人脸裁剪图、人体裁剪图、数据库、`.env`、服务器凭据或个人信息。

## 安装与更新

- Windows：下载并运行 `GKGuard-Setup-0.2.1.exe`。已安装旧版时，可使用软件右上角 `检查更新` 入口更新。
- macOS：下载 `GKGuard-macOS-*.dmg` 或 `GKGuard-macOS-*.zip`。当前仍是未签名、未公证的内部测试包。
- Linux：下载 `GKGuard-Linux-*.AppImage` 或 `GKGuard-Linux-*.deb`。
- Release 中应同时包含 `.blockmap`、`latest.yml`、`latest-mac.yml` 和 `latest-linux.yml`。

# GKGuard v0.2.1

## Summary

`v0.2.1` is a stability patch after the `v0.2.0` CampusVision C1 / GKGuard C2 feature integration. It does not change the main product entries. It addresses post-merge review findings around CampusVision C1 query-image cleanup, re-index idempotency, person-index and appearance-session rebuilds, person-attribute route ordering, route-to-record synchronization, and API specification examples, and it increments the GKGuard C2 frontend static asset version so installed apps do not reuse stale page resources.

## Highlights

- CampusVision C1 `POST /api/v1/query/face-image` now returns 400 for undecodable query images, returns 413 for query-image upload-size, decoded-pixel, or maximum-side violations, and cleans the temporary upload directory during these failure paths.
- Re-indexing the same CampusVision C1 `video_id` clears old events, person observations, face records, and the previous frame directory before replacing index artifacts, refreshes affected person-index entries, and rebuilds related appearance sessions so representative faces do not point at deleted face media and remaining events do not point at deleted sessions.
- GKGuard C2 sorts person-attribute route points independently by event time and preserves `recordIndex` / `eventId` mappings back to the score-ranked result records. If the default top-score record is not represented by a plotted route point, the result screen selects the first plotted route-point record by default, avoiding reversed or jumpy route start/end points, route screens with no active point, and wrong record selection after route clicks.
- GKGuard C2 route synchronization remains compatible with CampusVision C1 responses that omit `recordIndex` / `eventId`. Unmapped route points keep the current highlight by route index instead of collapsing the 6th and later route points into the last result record.
- The API specification now documents `/c1/query/face-image` controls correctly: `top_k`, `min_score`, `max_gap_sec`, candidate flags, event/match limits, and time filters are Query parameters, while `file` remains the multipart field.
- The person-attribute query example now uses `time_range.start_time` / `time_range.end_time`, matching the CampusVision C1 request model.
- The GKGuard C2 static asset parameter is updated to `v0.2.1-ui`; the desktop app continues to clear the renderer cache before loading the new CSS/JS.
- Collaboration-flow documents now clarify the PR-body reaction merge gate: only comment `@codex review` when there is no PR-body review signal and a review needs to be started actively; if the PR body already has 👀 or an equivalent review-in-progress signal, do not manually retrigger review and keep polling for the result.
- README, docs index, SECURITY, SUPPORT, CampusVision C1 / GKGuard C2 integration notes, auto-connection notes, demo script, and icon notes now describe `v0.2.1` as the current state.

## Behavior Boundary

- This release keeps the `v0.2.0` functional boundary: person-attribute search is explainable filtering and similar-event retrieval, not identity confirmation.
- Re-indexing a video replaces artifacts for that `video_id`, refreshes or deletes person entries that no longer have valid face records, and rebuilds appearance sessions for affected persons. If indexing fails, run indexing again rather than describing the failed state as preserving old event results.
- Upper color, glasses status, and appearance presentation still come from CampusVision C1 model/profile outputs and must not be treated as absolute facts.
- The repository does not commit real videos, real query images, extracted frames, face crops, body crops, databases, `.env`, server credentials, or personal information.

## Install And Update

- Windows: download and run `GKGuard-Setup-0.2.1.exe`. Existing installations can update through the top-right `检查更新` entry.
- macOS: download `GKGuard-macOS-*.dmg` or `GKGuard-macOS-*.zip`. The current package is still unsigned and unnotarized for internal testing.
- Linux: download `GKGuard-Linux-*.AppImage` or `GKGuard-Linux-*.deb`.
- The Release should also include `.blockmap`, `latest.yml`, `latest-mac.yml`, and `latest-linux.yml`.
