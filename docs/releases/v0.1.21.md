# GKGuard v0.1.21

## 更新重点

- 优化 GKGuard C2 结果页操作区：`重新上传`、`查看人物路线图`、`导出记录` 和 `定位记录列表` 的按钮布局、状态反馈和小窗口可用性更稳定。
- 优化 GKGuard C2 路线页：地图上方摘要、工具栏、路线图、时间线和关联记录在桌面、小窗口和窄屏下保持可读，路线图标签会避让边缘并使用紧凑图例。
- 统一检索、CampusVision C1 回退、定位、导出和更新入口的状态提示，减少同一流程中提示层级和按钮忙碌状态不一致的问题。
- 改进 CampusVision C1 连接提示：内嵌 SSH 密码窗口继续显示服务器账号、隧道目标、连接原因、进度、失败重试和密码安全说明，便于真实服务不可用时排障。
- 压缩移动端上传页表单和上传区高度，结果页和路线页操作按钮在窄屏下使用双列紧凑布局，降低 `390x720` 和 `680x640` 视口下首屏内容被挤出的问题。
- 同步 README、API 规范、CampusVision C1 / GKGuard C2 集成说明、演示脚本、支持/安全文档、Issue 模板、文档索引和版本号到 `v0.1.21`。

## 验证

- 通过：浏览器回归 `/demo?desktop=1`，覆盖 `1280x720`、`680x640` 和 `390x720` 视口。
- 通过：上传页进入、未上传图片本地模拟检索、结果页定位记录、打开人物路线图、定位时间线、返回结果页、重新上传回到上传页。
- 通过：三种视口均无页面级横向溢出，移动端开始检索按钮进入首屏，结果页和路线页窄屏按钮保持双列布局，路线图标签保持在视口内。
- 通过：浏览器 console warning/error 为 0，静态资源版本为 `uipolish5`。
- 通过：`python -m pytest backend`，25 passed，1 个上游 deprecation warning。
- 通过：`node --check backend/app/static/app.js`、`node --check desktop/main.js`、`node --check desktop/preload.js`。
- 通过：`python -m py_compile backend/desktop_server.py`。
- 通过：`npm audit --audit-level=low`，0 vulnerabilities。
- 通过：`npm run dist`，已生成 `GKGuard-Setup-0.1.21.exe` 和 `.blockmap`。
- 通过：`git diff --check`，仅 Windows CRLF 提示。

## 发布文件

- `GKGuard-Setup-0.1.21.exe`：Windows 桌面安装包。
- `GKGuard-Setup-0.1.21.exe.blockmap`：Electron 更新差分元数据。
- `latest.yml`：Electron 自动更新元数据。

普通首次安装下载并运行 `GKGuard-Setup-0.1.21.exe`。已安装旧版时，可使用软件右上角 `检查更新` 入口更新。

# GKGuard v0.1.21

## Highlights

- Improved the GKGuard C2 result-page action area so `重新上传`, `查看人物路线图`, `导出记录`, and `定位记录列表` have more stable layout, feedback, and small-window behavior.
- Improved the GKGuard C2 route page so the top summary, toolbar, route map, timeline, and related records remain readable on desktop, small-window, and narrow-screen viewports. Route-map labels avoid viewport edges and use a compact legend.
- Unified status feedback for search, CampusVision C1 fallback, locate, export, and update actions, reducing mismatches between toast severity and button busy state.
- Refined the CampusVision C1 connection prompt: the embedded SSH password window still shows the server account, tunnel target, connection reason, progress, retry-on-failure state, and password-safety note for real-service troubleshooting.
- Compacted the mobile upload form and upload area, and switched result and route action groups to a two-column compact layout on narrow screens, reducing first-viewport crowding at `390x720` and `680x640`.
- Synchronized README, API specification, CampusVision C1 / GKGuard C2 integration notes, demo script, support/security docs, Issue templates, documentation index, and package versions to `v0.1.21`.

## Verification

- Passed: browser regression on `/demo?desktop=1` at `1280x720`, `680x640`, and `390x720`.
- Passed: upload-page entry, mock search without an uploaded image, result-list locate, route-map open, timeline locate, return to results, and re-upload reset to the upload page.
- Passed: no page-level horizontal overflow in all three viewports; the mobile search button stays in the first viewport; result and route action groups keep the two-column narrow-screen layout; route-map labels remain inside the viewport.
- Passed: browser console warning/error count is 0, with static assets on `uipolish5`.
- Passed: `python -m pytest backend`, 25 passed with 1 upstream deprecation warning.
- Passed: `node --check backend/app/static/app.js`, `node --check desktop/main.js`, and `node --check desktop/preload.js`.
- Passed: `python -m py_compile backend/desktop_server.py`.
- Passed: `npm audit --audit-level=low`, 0 vulnerabilities.
- Passed: `npm run dist`, producing `GKGuard-Setup-0.1.21.exe` and `.blockmap`.
- Passed: `git diff --check`, with only Windows CRLF notices.

## Release Assets

- `GKGuard-Setup-0.1.21.exe`: Windows desktop installer.
- `GKGuard-Setup-0.1.21.exe.blockmap`: Electron differential update metadata.
- `latest.yml`: Electron auto-update metadata.

For a first-time installation, download and run `GKGuard-Setup-0.1.21.exe`. Existing installations can update through the top-right `检查更新` entry.
