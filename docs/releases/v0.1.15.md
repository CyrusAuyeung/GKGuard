# GKGuard v0.1.15

## 更新重点

- 修复输入服务器密码后已建立 SSH 隧道，但因 `/c1/status` 尚未及时选中 tunnel 而误提示“尚未检测到 CampusVision C1 服务”的问题。
- CampusVision C1 连接确认改为直接探测 `http://127.0.0.1:18000/openapi.json` 和 `/health`，只要 tunnel 端点可达即可继续真实检索。
- 将默认 CampusVision C1 连接等待从 45 秒缩短到 18 秒，并使用更短的分阶段探测间隔，减少启动和连接等待时间。
- 内嵌 SSH 密码窗口新增连接进度条和状态提示，显示“连接 SSH”“建立隧道”“检测 CampusVision C1”等阶段。
- README、API 规范、CampusVision C1 / GKGuard C2 集成说明、自动连接说明、演示脚本、安全支持范围、Issue 模板和发布说明已同步到 `v0.1.15` 当前状态。

## 使用说明

- 从 `v0.1.14` 升级到本版后，输入服务器密码时会留在软件内连接窗口并显示进度。
- 若 CampusVision C1 health 暂时慢于 OpenAPI 响应，只要 tunnel 已经能访问 CampusVision C1 端点，软件不会再弹出误导性的“未检测到 CampusVision C1 服务”提示。
- 如果仍然连接失败，请检查服务器密码、校园网/VPN、服务器端 CampusVision C1 服务和本机 18000 端口占用。

## 验证

- 通过：`python -m pytest backend`，24 passed，1 个上游 deprecation warning。
- 通过：`node --check desktop/main.js` 和 `node --check desktop/preload.js`。
- 通过：`npm ci` 和 `npm audit --audit-level=low`，0 vulnerabilities。
- 通过：`npm run dist`，已生成 `GKGuard-Setup-0.1.15.exe` 和 `.blockmap`。

## 发布文件

- `GKGuard-Setup-0.1.15.exe`：Windows 桌面安装包。
- `GKGuard-Setup-0.1.15.exe.blockmap`：Electron 更新差分元数据。
- `latest.yml`：Electron 自动更新元数据。

普通首次安装仍下载并运行 `GKGuard-Setup-0.1.15.exe`。

# GKGuard v0.1.15

## Highlights

- Fixed a false “CampusVision C1 service not detected” warning after a correct server password established the SSH tunnel but `/c1/status` had not selected the tunnel yet.
- CampusVision C1 connection confirmation now probes `http://127.0.0.1:18000/openapi.json` and `/health` directly, so real search can continue as soon as the tunnel endpoint is reachable.
- The default CampusVision C1 connection wait was reduced from 45 seconds to 18 seconds with shorter phased probing, reducing startup and connection wait time.
- The embedded SSH password window now includes a progress bar and status text for SSH connection, tunnel setup, and CampusVision C1 probing stages.
- README, API specification, CampusVision C1 / GKGuard C2 integration notes, auto-connection guide, demo script, supported security version, issue templates, and Release notes are synced to the current `v0.1.15` state.

## Usage Notes

- After upgrading from `v0.1.14`, entering the server password stays inside the app connection window and shows progress.
- If CampusVision C1 health is slower than OpenAPI but the tunnel can already reach a CampusVision C1 endpoint, the app no longer shows the misleading “CampusVision C1 service not detected” warning.
- If connection still fails, check the server password, campus network/VPN, server-side CampusVision C1 process, and local port 18000 usage.

## Verification

- Passed: `python -m pytest backend`, 24 passed with 1 upstream deprecation warning.
- Passed: `node --check desktop/main.js` and `node --check desktop/preload.js`.
- Passed: `npm ci` and `npm audit --audit-level=low`, 0 vulnerabilities.
- Passed: `npm run dist`, producing `GKGuard-Setup-0.1.15.exe` and `.blockmap`.

## Release Assets

- `GKGuard-Setup-0.1.15.exe`: Windows desktop installer.
- `GKGuard-Setup-0.1.15.exe.blockmap`: Electron differential update metadata.
- `latest.yml`: Electron auto-update metadata.

For a first-time installation, download and run `GKGuard-Setup-0.1.15.exe`.
