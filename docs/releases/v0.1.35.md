# GKGuard v0.1.35

## 变更摘要

`v0.1.35` 是一次集中安全加固版本，基于当前 `main` 重新整合并验证安全扫描生成的多项修复。版本继续保留 `v0.1.34` 的 Windows/macOS/Linux 桌面端发布能力，同时收紧 GKGuard C2 本地后端、CampusVision C1 代理、CampusVision C1 服务端上传处理、审计/案件包接口和 Release workflow 权限边界。

## 主要更新

- GKGuard C2 桌面后端固定绑定 loopback，避免桌面本地服务被环境变量误配置为外部监听。
- CampusVision C1 默认候选只保留本机 SSH 隧道；校园网直连地址必须显式配置，并通过 `C1_ALLOWED_HOSTS`、OpenAPI 身份检查和 `/health` 检查后才会接收请求。
- Electron 内嵌 SSH 隧道在发送密码前校验主机密钥，支持固定 `SHA256:...` 指纹或首次连接人工确认，并避免 SSH `ECONNRESET` 弹出主进程 JavaScript 错误。
- CampusVision C1 在绑定外部地址或显式要求时启用 API key 校验，上传、索引、人物库和检索接口会在读取 multipart 请求体前检查 `X-CampusVision-API-Key`。
- CampusVision C1 查询图、视频上传、图像解码和视频索引增加资源上限；超限请求返回结构化错误，并清理已经保存的临时文件或部分索引结果。
- GKGuard C2 旧版模拟图片搜索增加 2 MiB 文件上限和请求体大小预检。
- 审计日志读取和案件包导出增加本地保护 token，审计日志按字段长度、行数和字节数压缩保留，避免无限增长。
- 前端避免把 CampusVision C1 返回的图片 URL 或时间字段写入危险 DOM sink。
- Release workflow 使用更小的默认权限，将 Release 写权限限定在发布 job，并改用固定版本的 artifact 下载与 GitHub CLI 发布流程。

## 安装与更新

- Windows：下载并运行 `GKGuard-Setup-0.1.35.exe`。已安装旧版时，可使用软件右上角 `检查更新` 入口更新。
- macOS：下载 `GKGuard-macOS-*.dmg` 或 `GKGuard-macOS-*.zip`。当前仍是未签名、未公证的内部测试包。
- Linux：下载 `GKGuard-Linux-*.AppImage` 或 `GKGuard-Linux-*.deb`。
- Release 中应同时包含 `.blockmap`、`latest.yml`、`latest-mac.yml` 和 `latest-linux.yml`。

## 运维注意

- 如果 CampusVision C1 仍只绑定服务器本机 `127.0.0.1:8000`，推荐继续使用本机 SSH 隧道 `127.0.0.1:18000`。
- 如果 CampusVision C1 需要校园网直连，必须显式配置候选 URL、`C1_ALLOWED_HOSTS` 和 CampusVision C1 API key，不要把密钥提交到仓库。
- 读取 `/audit/logs` 需要 `GKGUARD_AUDIT_TOKEN` 与 `X-GKGuard-Audit-Token`。
- 导出 `/events/{event_id}/case-package` 需要 `GKGUARD_CASE_PACKAGE_EXPORT_TOKEN` 与 `X-GKGuard-Export-Token`。

# GKGuard v0.1.35

## Summary

`v0.1.35` is a consolidated security-hardening release that re-implements and validates the security-scan fixes against the current `main` branch. It preserves the Windows/macOS/Linux desktop release capability from `v0.1.34` while tightening the GKGuard C2 local backend, CampusVision C1 proxy, CampusVision C1 upload handling, audit/case-package endpoints, and Release workflow permissions.

## Highlights

- The GKGuard C2 desktop backend is fixed to loopback binding so local desktop services are not accidentally exposed by environment variables.
- The default CampusVision C1 candidate list keeps only the local SSH tunnel. Campus-network direct URLs must be explicitly configured and must pass `C1_ALLOWED_HOSTS`, OpenAPI identity checks, and `/health` checks before receiving requests.
- Electron embedded SSH verifies the host key before sending the password, supports pinned `SHA256:...` fingerprints or first-use manual confirmation, and prevents SSH `ECONNRESET` from surfacing as a main-process JavaScript error dialog.
- CampusVision C1 enforces API-key validation when bound externally or explicitly configured to do so; upload, indexing, person-index, and search endpoints validate `X-CampusVision-API-Key` before reading multipart request bodies.
- CampusVision C1 adds resource limits for query images, video uploads, image decoding, and video indexing; over-limit requests return structured errors and clean up saved temporary files or partial indexing results.
- Legacy GKGuard C2 mock image search adds a 2 MiB file limit and request-size preflight.
- Audit-log readback and case-package export require local protection tokens, and audit logs are compacted by field length, line count, and byte count to avoid unbounded growth.
- The frontend avoids writing CampusVision C1-returned image URLs or time fields into dangerous DOM sinks.
- The Release workflow uses smaller default permissions, limits Release write permissions to the publish job, and uses pinned artifact-download plus GitHub CLI release publishing.

## Install And Update

- Windows: download and run `GKGuard-Setup-0.1.35.exe`. Existing installations can update through the top-right `检查更新` entry.
- macOS: download `GKGuard-macOS-*.dmg` or `GKGuard-macOS-*.zip`. The current package is still unsigned and unnotarized for internal testing.
- Linux: download `GKGuard-Linux-*.AppImage` or `GKGuard-Linux-*.deb`.
- The Release should also include `.blockmap`, `latest.yml`, `latest-mac.yml`, and `latest-linux.yml`.

## Operations Notes

- If CampusVision C1 still binds only to the server-local `127.0.0.1:8000`, keep using the local SSH tunnel `127.0.0.1:18000`.
- If CampusVision C1 needs campus-network direct access, explicitly configure candidate URLs, `C1_ALLOWED_HOSTS`, and the CampusVision C1 API key. Do not commit keys to the repository.
- Reading `/audit/logs` requires `GKGUARD_AUDIT_TOKEN` and `X-GKGuard-Audit-Token`.
- Exporting `/events/{event_id}/case-package` requires `GKGUARD_CASE_PACKAGE_EXPORT_TOKEN` and `X-GKGuard-Export-Token`.

- `python -m py_compile backend/desktop_server.py`
