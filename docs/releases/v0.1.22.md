# GKGuard v0.1.22

## 更新重点

- 继续精修 GKGuard C2 移动端结果页和路线页：主要操作按钮保持双列布局，并提升到约 `42px` 触控高度，减少窄屏误触和文字挤压。
- 为结果页和路线页记录列表增加横向滚动提示、右侧渐隐和滚动吸附，让可横滑内容更容易被发现。
- 优化 GKGuard C2 移动端路线页信息层级：在地图前新增当前轨迹摘要，展示当前记录、出现时间、位置和相似度；完整关联记录仍保留在页面下方。
- 修复 `390x720` 视口下本地模拟关键帧装饰和路线地图元素的边界问题；真实 CampusVision C1 关键帧仍保持完整显示，不裁切主体画面。
- 修复状态提示初始、切屏和重置时可能出现空白 toast 的问题，并保留检索、CampusVision C1 回退、定位、导出和更新入口的统一提示逻辑。
- 将静态资源缓存版本从临时 `uipolish5` 命名切换为稳定的 `v0.1.22-ui`，避免桌面壳复用旧后台时加载过期页面资源。
- 同步 README、API 规范、CampusVision C1 / GKGuard C2 集成说明、演示脚本、支持/安全文档、Issue 模板、文档索引和版本号到 `v0.1.22`。

## 验证

- 通过：浏览器回归 `/demo?desktop=1`，覆盖 `1280x720`、`680x640` 和 `390x720` 视口。
- 通过：上传页进入、未上传图片本地模拟检索、结果页定位记录、打开人物路线图、定位时间线、返回结果页、重新上传回到上传页。
- 通过：三种视口均无页面级横向溢出、无按钮文字溢出、无空白 toast，浏览器 console warning/error 为 0。
- 通过：移动端结果页和路线页按钮保持双列触控高度，记录列表显示横向滑动提示，路线页当前轨迹摘要位于地图前。
- 通过：`python -m pytest backend`，25 passed，1 个上游 deprecation warning。
- 通过：`node --check backend/app/static/app.js`、`node --check desktop/main.js`、`node --check desktop/preload.js`。
- 通过：`python -m py_compile backend/desktop_server.py`。
- 通过：`npm audit --audit-level=low`，0 vulnerabilities。
- 通过：`npm run dist`，已生成 `GKGuard-Setup-0.1.22.exe` 和 `.blockmap`。
- 通过：`git diff --check`，仅 Windows CRLF 提示。

## 发布文件

- `GKGuard-Setup-0.1.22.exe`：Windows 桌面安装包。
- `GKGuard-Setup-0.1.22.exe.blockmap`：Electron 更新差分元数据。
- `latest.yml`：Electron 自动更新元数据。

普通首次安装下载并运行 `GKGuard-Setup-0.1.22.exe`。已安装旧版时，可使用软件右上角 `检查更新` 入口更新。

# GKGuard v0.1.22

## Highlights

- Continued mobile polish for the GKGuard C2 result and route screens: primary action groups keep the two-column layout and use about `42px` touch height to reduce cramped text and mis-taps on narrow screens.
- Added horizontal-scroll hints, right-edge fades, and scroll snapping to result and route record lists so horizontally scrollable content is easier to discover.
- Improved the GKGuard C2 mobile route-screen hierarchy with a current-trajectory summary before the map, showing the selected record, time, location, and similarity. The full related-record list remains lower on the page.
- Fixed boundary issues for local mock keyframe decoration and route-map elements at `390x720`; real CampusVision C1 keyframes still render fully without cropping the subject.
- Fixed empty toast states during initialization, screen switches, and reset flows while preserving the unified feedback logic for search, CampusVision C1 fallback, locate, export, and update actions.
- Replaced the temporary `uipolish5` static asset cache key with the stable `v0.1.22-ui` naming so the desktop shell does not reuse stale page resources from an older backend.
- Synchronized README, API specification, CampusVision C1 / GKGuard C2 integration notes, demo script, support/security docs, Issue templates, documentation index, and package versions to `v0.1.22`.

## Verification

- Passed: browser regression on `/demo?desktop=1` at `1280x720`, `680x640`, and `390x720`.
- Passed: upload-page entry, mock search without an uploaded image, result-list locate, route-map open, timeline locate, return to results, and re-upload reset to the upload page.
- Passed: no page-level horizontal overflow, button text overflow, or empty toast in all three viewports; browser console warning/error count is 0.
- Passed: mobile result and route action groups keep two-column touch-height layout, record lists show horizontal-scroll hints, and the route current-trajectory summary appears before the map.
- Passed: `python -m pytest backend`, 25 passed with 1 upstream deprecation warning.
- Passed: `node --check backend/app/static/app.js`, `node --check desktop/main.js`, and `node --check desktop/preload.js`.
- Passed: `python -m py_compile backend/desktop_server.py`.
- Passed: `npm audit --audit-level=low`, 0 vulnerabilities.
- Passed: `npm run dist`, producing `GKGuard-Setup-0.1.22.exe` and `.blockmap`.
- Passed: `git diff --check`, with only Windows CRLF notices.

## Release Assets

- `GKGuard-Setup-0.1.22.exe`: Windows desktop installer.
- `GKGuard-Setup-0.1.22.exe.blockmap`: Electron differential update metadata.
- `latest.yml`: Electron auto-update metadata.

For a first-time installation, download and run `GKGuard-Setup-0.1.22.exe`. Existing installations can update through the top-right `检查更新` entry.
