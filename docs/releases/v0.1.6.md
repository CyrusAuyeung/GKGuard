# GKGuard v0.1.6

## 中文

## 重点更新

- 新增 CampusCar / UE 集成合同，明确 C2 只保留现场复核与外部控制系统的接口边界。
- 补充 CampusCar mock dispatch 和 UE bridge readiness 的占位说明，方便后续对接控制组、UE 和 ROS2 服务。
- 明确 C2 不打包 UE 运行时，不直接运行 ROS2 节点，也不接管底盘控制。
- 延续完整桌面壳、C2 mock 演示和 Windows 安装包发布流程。

## 安装与网络说明

- 本版本可打开 C2 桌面壳和本地 mock 演示。
- C2 mock 演示不依赖校园网；CampusCar/UE 仍为占位合同，不会连接真实车辆或 UE 服务。
- 真实 C1、校园视频或服务器检索能力在该版本尚未接入；如需当前 C1 真实检索，请使用 v0.1.8 或更新版本。

## 验证情况

- GitHub Actions 成功构建 Windows 安装包。
- 已验证 CampusCar/UE 占位接口和桌面壳演示路径。

## 安装包

本 Release 下方包含：

- `GKGuard-Setup-0.1.6.exe`：Windows 桌面安装包。
- `GKGuard-Setup-0.1.6.exe.blockmap`：Electron 更新差分元数据。
- `latest.yml`：Electron 更新通道元数据。

普通试用建议下载 `.exe` 文件；`.blockmap` 和 `latest.yml` 通常不需要手动打开。

# GKGuard v0.1.6

## English

## Highlights

- Added the CampusCar / UE integration contract, defining C2's boundary for field review and external control-system handoff.
- Added placeholder documentation for CampusCar mock dispatch and UE bridge readiness, preparing later integration with the control, UE, and ROS2 services.
- Clarified that C2 does not package the UE runtime, does not run ROS2 nodes directly, and does not own chassis control.
- Continued the complete desktop shell, C2 mock demo, and Windows installer release flow.

## Installation And Network Notes

- This version can open the C2 desktop shell and local mock demo.
- The C2 mock demo does not require campus network access; CampusCar/UE remains a placeholder contract and does not connect to a real vehicle or UE service.
- Real C1, campus video, or server-side search was not integrated in this version. Use v0.1.8 or later for the current real C1 search integration.

## Verification

- GitHub Actions successfully built the Windows installer.
- Verified CampusCar/UE placeholder APIs and desktop-shell demo paths.

## Installer Assets

This Release includes:

- `GKGuard-Setup-0.1.6.exe`: Windows desktop installer.
- `GKGuard-Setup-0.1.6.exe.blockmap`: Electron differential update metadata.
- `latest.yml`: Electron update-channel metadata.

For normal testing, download the `.exe` file; `.blockmap` and `latest.yml` usually do not need to be opened manually.
