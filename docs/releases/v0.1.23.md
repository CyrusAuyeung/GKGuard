# GKGuard v0.1.23

## 中文

## 概要

- 新增 Playwright 浏览器 E2E 回归，覆盖 `1280x720`、`680x640` 和 `390x720` 视口下的本地模拟检索、路线联动、重新上传，以及模拟 CampusVision C1 媒体结果的关键帧预览弹窗。
- 精修 GKGuard C2 前端细节：结果页关键帧可打开预览弹窗，路线图点位、关联记录和时间线可联动定位当前记录，并补充空状态和键盘焦点状态。
- 新增 `scripts/Update-RoadmapItem.ps1`，可半自动把 PR/Issue 加入 GKGuard Roadmap 并填写 `Status`、`Area`、`Type`、`Priority`、`Blocked`、`Start date`、`End date`、`Timeline order` 和 `Target version` 字段。
- 同步 README、API 规范、CampusVision C1 / GKGuard C2 集成说明、演示脚本、支持/安全文档、贡献指南、PR 模板、文档索引和版本号到 `v0.1.23`。
- 将静态资源缓存版本更新为 `v0.1.23-ui`，避免桌面壳复用旧后台时加载过期页面资源。

## 验证

- 通过：`python -m pytest backend`，25 passed，1 个上游 deprecation warning。
- 通过：`node --check backend/app/static/app.js`、`node --check desktop/main.js`、`node --check desktop/preload.js`。
- 通过：`python -m py_compile backend/desktop_server.py`。
- 通过：`npm run test:e2e`，6 个 Chromium 用例通过。
- 通过：`npm audit --audit-level=low`，0 vulnerabilities。
- 通过：`git diff --check`，仅 Windows 换行提示。
- 通过：`scripts/Update-RoadmapItem.ps1` 使用既有 PR item 的同值写入回归检查。
- 通过：`npm run dist`，已生成 `GKGuard-Setup-0.1.23.exe` 和 `.blockmap`。

## 发布文件

- `GKGuard-Setup-0.1.23.exe`：Windows 桌面安装包。
- `GKGuard-Setup-0.1.23.exe.blockmap`：Electron 更新差分元数据。
- `latest.yml`：Electron 自动更新索引。

## 使用说明

普通首次安装下载并运行 `GKGuard-Setup-0.1.23.exe`。已安装旧版时，可使用软件右上角 `检查更新` 入口更新。

软件本体和本地模拟回退不依赖校园网；真实 CampusVision C1 检索仍需要校园网、VPN 或 SSH 隧道访问 CampusVision C1 服务所在服务器。

# GKGuard v0.1.23

## English

## Summary

- Added Playwright browser E2E regression across `1280x720`, `680x640`, and `390x720`, covering mock search, route selection sync, re-upload, and a mocked CampusVision C1 media keyframe preview dialog.
- Polished GKGuard C2 frontend details: result keyframes can open a preview dialog, route-map points, related records, and timeline rows synchronize with the current record, and empty plus keyboard focus states are now explicit.
- Added `scripts/Update-RoadmapItem.ps1` to semi-automatically add PRs/Issues to GKGuard Roadmap and fill `Status`, `Area`, `Type`, `Priority`, `Blocked`, `Start date`, `End date`, `Timeline order`, and `Target version`.
- Synchronized README, API specification, CampusVision C1 / GKGuard C2 integration notes, demo script, support/security docs, contributing guide, PR template, documentation index, and package versions to `v0.1.23`.
- Updated the static asset cache key to `v0.1.23-ui` so the desktop shell does not reuse stale page resources from an older backend.

## Validation

- Passed: `python -m pytest backend`, 25 passed with 1 upstream deprecation warning.
- Passed: `node --check backend/app/static/app.js`, `node --check desktop/main.js`, and `node --check desktop/preload.js`.
- Passed: `python -m py_compile backend/desktop_server.py`.
- Passed: `npm run test:e2e`, 6 Chromium cases.
- Passed: `npm audit --audit-level=low`, 0 vulnerabilities.
- Passed: `git diff --check`, with Windows line-ending warnings only.
- Passed: `scripts/Update-RoadmapItem.ps1` same-value regression check against an existing PR Project item.
- Passed: `npm run dist`, producing `GKGuard-Setup-0.1.23.exe` and `.blockmap`.

## Release Assets

- `GKGuard-Setup-0.1.23.exe`: Windows desktop installer.
- `GKGuard-Setup-0.1.23.exe.blockmap`: Electron differential update metadata.
- `latest.yml`: Electron auto-update index.

## Usage

For a first-time installation, download and run `GKGuard-Setup-0.1.23.exe`. Existing installations can update through the top-right `检查更新` entry.

The app itself and mock fallback do not require campus network access; real CampusVision C1 search still requires campus network, VPN, or an SSH tunnel to the CampusVision C1 service host.
