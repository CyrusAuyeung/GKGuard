# GKGuard v0.1.24

## 重点更新

- 新增查询图人脸检测接口链路：GKGuard C2 通过 `/c1/query-faces` 调用 CampusVision C1 `/api/v1/search/query-faces`，返回查询图内人脸框和检测置信度。
- 优化上传检索体验：单人图片自动选中并直接检索；多人图片在原图上显示人脸框和检测置信度，用户选择目标后再检索。
- 新增选中人脸以图搜人参数：GKGuard C2 在需要时向 `/c1/search/person-by-image` 传递 `query_face_index`，CampusVision C1 只使用选中查询人脸的 embedding 检索。
- 结果详情和关键帧预览新增目标人脸框：CampusVision C1 命中帧 `bbox` 会映射为 `records[].faceBox`，GKGuard C2 在关键帧和弹窗中标出目标人脸并显示相似度。
- 同步 README、API 规范、CampusVision C1 / GKGuard C2 集成说明、数据字典、演示脚本、CampusVision C1 子服务文档、支持/安全文档、文档索引和版本号到 `v0.1.24`。
- 将静态资源缓存版本更新为 `v0.1.24-ui`，避免桌面壳复用旧后台时加载过期页面资源。

## 行为说明

- 查询图片未检测到人脸时，GKGuard C2 停留在上传页并提示重新上传清晰正脸照片，不进入结果页。
- 查询图片只有一张人脸时，GKGuard C2 自动选择该人脸并调用真实检索。
- 查询图片有多张人脸时，GKGuard C2 不会默认任选一个目标，而是在原图上要求用户明确选择。
- 检测置信度只表示查询图人脸检测质量；结果页相似度仍表示目标人脸与 CampusVision C1 人物库/关键帧的匹配分数。
- 本版本不向仓库加入真实监控、人脸或个人信息数据。

## 验证

- 通过：`python -m pytest backend`，`26 passed`。
- 通过：`node --check backend/app/static/app.js`。
- 通过：`npm run test:e2e`，9 个 Chromium 用例覆盖 `1280x720`、`680x640` 和 `390x720` 下的本地模拟、单人自动检索、多人框选、结果关键帧框选和预览弹窗。
- 通过：`npm run dist`，已生成 `GKGuard-Setup-0.1.24.exe`、`GKGuard-Setup-0.1.24.exe.blockmap` 和 `latest.yml`。
- 发布前仍需在 GitHub Actions 中确认 `Verify` 通过，并在正式 GitHub Release 中核对安装包、`.blockmap` 和 `latest.yml` 齐全。

## 安装

普通首次安装下载并运行 `GKGuard-Setup-0.1.24.exe`。已安装旧版时，可使用软件右上角 `检查更新` 入口更新。

真实 CampusVision C1 检索仍需要校园网、VPN 或 SSH 隧道能访问 CampusVision C1 服务；离线环境会回退本地模拟流程。

# GKGuard v0.1.24

## Highlights

- Added the query-face detection path: GKGuard C2 calls CampusVision C1 `/api/v1/search/query-faces` through `/c1/query-faces` and receives face boxes plus detection confidence for the uploaded query image.
- Improved upload search behavior: single-face images are auto-selected and searched directly; multi-face images show face boxes and detection confidence on the original upload so the user selects the target before search.
- Added selected-face search: GKGuard C2 passes `query_face_index` to `/c1/search/person-by-image` when needed, and CampusVision C1 searches with only the selected query-face embedding.
- Added target-face overlays for result details and keyframe preview: CampusVision C1 matched-frame `bbox` maps to `records[].faceBox`, and GKGuard C2 overlays the target face with similarity on keyframes and dialogs.
- Synchronized README, API specification, CampusVision C1 / GKGuard C2 integration notes, data dictionary, demo script, CampusVision C1 service docs, support/security docs, documentation index, and package versions to `v0.1.24`.
- Updated the static asset cache key to `v0.1.24-ui` so the desktop shell does not reuse stale page resources from an older backend.

## Behavior Notes

- If no face is detected in the query image, GKGuard C2 stays on the upload screen and asks for a clearer face image instead of entering results.
- If exactly one face is detected, GKGuard C2 auto-selects that face and runs real search.
- If multiple faces are detected, GKGuard C2 does not pick a default target; it requires the user to choose the intended face on the original image.
- Detection confidence describes query-face detection quality only; result similarity still describes the match between the target face and CampusVision C1 person/keyframe records.
- This release does not add real surveillance, face, or personal-information data to the repository.

## Verification

- Passed: `python -m pytest backend`, `26 passed`.
- Passed: `node --check backend/app/static/app.js`.
- Passed: `npm run test:e2e`, with 9 Chromium cases covering local mock flow, single-face auto-search, multi-face selection, result keyframe overlays, and preview dialog behavior at `1280x720`, `680x640`, and `390x720`.
- Passed: `npm run dist`, producing `GKGuard-Setup-0.1.24.exe`, `GKGuard-Setup-0.1.24.exe.blockmap`, and `latest.yml`.
- Before publishing, still confirm GitHub Actions `Verify` passes and the official GitHub Release contains the installer, `.blockmap`, and `latest.yml`.

## Installation

For a first-time installation, download and run `GKGuard-Setup-0.1.24.exe`. Existing installations can update through the top-right `检查更新` entry.

Real CampusVision C1 search still requires campus network, VPN, or an SSH tunnel to the CampusVision C1 service. Offline environments fall back to the local mock flow.
