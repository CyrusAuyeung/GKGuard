# GKGuard v0.1.31

## 重点更新

- 修复结果页左侧人物照片裁切偏移：人物照片现在直接使用用户在上传区或放大选人弹窗中点击的目标人脸框作为裁切坐标来源。
- 修复 CampusVision C1 返回像素 bbox 且附带 `imageWidth` / `imageHeight` 时的坐标换算：前端会先按 CampusVision C1 源图尺寸映射到浏览器实际图片尺寸，再生成目标人脸裁切图。
- 保留查询图人脸框的 CampusVision C1 源图尺寸字段，确保上传区、放大弹窗和结果页人物照片使用同一套坐标语义。
- 补充 E2E 回归：使用合成双色图片验证“选中右侧候选人脸后，结果页人物照片也裁到右侧区域”，避免只验证图片存在而没有验证裁切内容。
- 刷新 `package-lock.json` 中 `node-gyp` 下游 `undici` 到 `6.27.0`，消除当前 `npm audit --audit-level=low` 报告的高危漏洞。
- 同步 README、API 规范、CampusVision C1 / GKGuard C2 集成说明、自动连接说明、演示脚本、文档索引、图标说明和版本号到 `v0.1.31`。

## 行为说明

- 多人图选择阶段的框位置和结果页人物照片现在以同一个可见目标框为准；如果用户在放大弹窗里选择目标，结果页左侧人物照片应展示该目标框内的人脸，而不是偏到相邻人物或图像边缘。
- 本版本不修改远端 CampusVision C1 服务，不改变相似度计算和检索排序逻辑，也不向仓库加入真实监控、人脸或个人信息数据。

## 验证

- 通过：`node --check backend/app/static/app.js`。
- 通过：`node --check tests/e2e/gkguard-ui.spec.js`。
- 通过：`python -m py_compile backend/desktop_server.py`。
- 通过：`python -m pytest backend`，结果为 `27 passed`。
- 通过：`python -m pytest services/campusvision-c1/tests`，本地缺少 OpenCV 相关依赖时按测试级跳过 `2 skipped`。
- 通过：`npm run test:e2e`，覆盖 18 个 Chromium 用例。
- 通过：`npm audit --audit-level=low`，结果为 `0 vulnerabilities`。
- 通过：`git diff --check`。
- 通过：`npm run dist`，本地生成 `GKGuard-Setup-0.1.31.exe` 和 `.blockmap`。
- 发布流程要求 GitHub Actions `Verify` 和 Release workflow 通过，并在正式 GitHub Release 中核对安装包、`.blockmap` 和 `latest.yml` 齐全。

## 安装

普通首次安装下载并运行 `GKGuard-Setup-0.1.31.exe`。已安装旧版时，可使用软件右上角 `检查更新` 入口更新。

真实 CampusVision C1 检索仍需要校园网、VPN 或 SSH 隧道能访问 CampusVision C1 服务；未上传图片时可使用本地模拟流程演示，上传图片后的检测失败、无匹配、超时或真实检索失败不会展示模拟命中。

# GKGuard v0.1.31

## Highlights

- Fixed result-side portrait crop drift: the portrait now crops from the exact target face box clicked in the upload area or enlarged face-selection modal.
- Fixed coordinate conversion when CampusVision C1 returns pixel bboxes with `imageWidth` / `imageHeight`: the frontend maps CampusVision C1 source-image coordinates to the browser image before generating the target-face crop.
- Preserved CampusVision C1 source-image dimensions on query-face boxes so the upload area, enlarged modal, and result portrait share the same coordinate semantics.
- Added an E2E regression using a synthetic two-color image to verify that selecting the right-side candidate also crops the right-side region in the result portrait, instead of only checking that an image exists.
- Refreshed the `package-lock.json` `node-gyp` transitive `undici` dependency to `6.27.0`, clearing the current high-severity `npm audit --audit-level=low` report.
- Synchronized README, API specification, CampusVision C1 / GKGuard C2 integration notes, auto-connection guide, demo script, documentation index, icon notes, and package versions to `v0.1.31`.

## Behavior Notes

- Multi-face selection boxes and the result portrait now use the same visible target-box source. If the user selects a target in the enlarged modal, the left-side result portrait should show the face inside that box rather than drifting to a neighboring person or image edge.
- This release does not modify the remote CampusVision C1 service, does not change similarity scoring or result ranking, and does not add real surveillance, face, or personal-information data to the repository.

## Verification

- Passed: `node --check backend/app/static/app.js`.
- Passed: `node --check tests/e2e/gkguard-ui.spec.js`.
- Passed: `python -m py_compile backend/desktop_server.py`.
- Passed: `python -m pytest backend`, with `27 passed`.
- Passed: `python -m pytest services/campusvision-c1/tests`, with `2 skipped` locally when OpenCV-related dependencies are unavailable.
- Passed: `npm run test:e2e`, covering 18 Chromium cases.
- Passed: `npm audit --audit-level=low`, with `0 vulnerabilities`.
- Passed: `git diff --check`.
- Passed: `npm run dist`, generating `GKGuard-Setup-0.1.31.exe` and `.blockmap` locally.
- Release flow requires GitHub Actions `Verify` and the Release workflow to pass, followed by checking that the official GitHub Release contains the installer, `.blockmap`, and `latest.yml`.

## Installation

For a first-time installation, download and run `GKGuard-Setup-0.1.31.exe`. Existing installations can update through the top-right `检查更新` entry.

Real CampusVision C1 search still requires campus network, VPN, or an SSH tunnel to the CampusVision C1 service. The mock flow remains available when no image has been uploaded; uploaded-image detection failures, no-match results, timeouts, or real-search failures do not show mock hits.
