# GKGuard v0.1.7

## 中文

## 重点更新

- 导入 CampusVision C1 服务源码快照，作为后续 C2 真实视频检索接入的服务基础和参考材料。
- 重建人脸检索 demo UI，形成 `人脸检索`、`人脸检索结果`、`人物路线图` 三屏主流程。
- 修复 Release notes 状态插值问题，减少自动生成说明中的格式异常。
- 将桌面包版本推进到 `0.1.7` 并发布 Windows 安装包。

## 安装与网络说明

- 本版本可打开 C2 桌面壳和三屏人脸检索 demo 壳体。
- C1 源码已经进入仓库，但 C2 在 v0.1.7 仍未通过 `/c1/...` 代理连接真实 C1 服务。
- 因此本版本 mock 演示不依赖校园网；真实 C1 检索请使用 v0.1.8 或更新版本，并准备 C1 服务器访问方式。

## 验证情况

- GitHub Actions 成功构建 Windows 安装包。
- 已验证 C1 源码快照进入仓库、三屏前端壳体和 Release note 修复提交。

## 安装包

本 Release 下方包含：

- `GKGuard-Setup-0.1.7.exe`：Windows 桌面安装包。
- `GKGuard-Setup-0.1.7.exe.blockmap`：Electron 更新差分元数据。
- `latest.yml`：Electron 更新通道元数据。

普通试用建议下载 `.exe` 文件；`.blockmap` 和 `latest.yml` 通常不需要手动打开。

# GKGuard v0.1.7

## English

## Highlights

- Imported the CampusVision C1 service source snapshot as the service foundation and reference material for later real video-search integration.
- Rebuilt the face-search demo UI into the three-screen flow: face search, search results, and person route map.
- Fixed Release notes status interpolation to reduce formatting issues in generated notes.
- Advanced the desktop package version to `0.1.7` and published the Windows installer.

## Installation And Network Notes

- This version can open the C2 desktop shell and the three-screen face-search demo shell.
- The C1 source is in the repository, but C2 in v0.1.7 does not yet connect to the real C1 service through `/c1/...` proxy endpoints.
- Therefore the mock demo does not require campus network access. Use v0.1.8 or later for real C1 search, with access to the C1 server prepared.

## Verification

- GitHub Actions successfully built the Windows installer.
- Verified that the C1 source snapshot, three-screen frontend shell, and Release-note fix were included.

## Installer Assets

This Release includes:

- `GKGuard-Setup-0.1.7.exe`: Windows desktop installer.
- `GKGuard-Setup-0.1.7.exe.blockmap`: Electron differential update metadata.
- `latest.yml`: Electron update-channel metadata.

For normal testing, download the `.exe` file; `.blockmap` and `latest.yml` usually do not need to be opened manually.
