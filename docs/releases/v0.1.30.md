# GKGuard v0.1.30

## 重点更新

- 修复多人选择弹窗默认尺寸：弹窗会先显示再计算图片适应比例，默认和 `适应` 按可见容器内最大无滚动尺寸完整展示原图，不再回到过小尺寸。
- 扩大多人选择弹窗桌面端可用空间：弹窗 surface 使用固定可用高度和更宽的桌面上限，让大图选择更接近实际窗口可用区域。
- 修复结果页人物照片来源：人物照片优先显示选人阶段目标框内的查询人脸裁切图，缺失时才回退 CampusVision C1 返回的人脸缩略图。
- 修复查询人脸裁切边界：本地裁切直接使用选中目标框 bbox，不再额外外扩；极小测试图也不会生成空 `data:,` 图片。
- 同步 README、API 规范、CampusVision C1 / GKGuard C2 集成说明、自动连接说明、演示脚本、文档索引、图标说明和版本号到 `v0.1.30`。

## 行为说明

- 多人图弹窗的默认态和 `适应` 按“长宽两维都能完整显示且不需要滚动”的最大尺寸计算；用户仍可继续缩小或放大做精细选择。
- 结果页左侧人物照片用于展示本次检索目标，因此以用户在上传图中选中的目标框裁切为准；检索记录缩略图仍优先使用 CampusVision C1 返回的人脸缩略图。
- 本版本不修改远端 CampusVision C1 服务，不向仓库加入真实监控、人脸或个人信息数据。

## 验证

- 通过：`node --check backend/app/static/app.js`。
- 通过：`python -m pytest backend`。
- 通过：`npm run test:e2e`，覆盖多人查询图弹窗最大无滚动适应、缩小后 `适应` 恢复最大完整显示、结果页人物照片使用选中查询人脸裁切、关键帧目标框、无匹配/超时恢复和响应式布局。
- 发布流程要求 GitHub Actions `Verify` 和 Release workflow 通过，并在正式 GitHub Release 中核对安装包、`.blockmap` 和 `latest.yml` 齐全。

## 安装

普通首次安装下载并运行 `GKGuard-Setup-0.1.30.exe`。已安装旧版时，可使用软件右上角 `检查更新` 入口更新。

真实 CampusVision C1 检索仍需要校园网、VPN 或 SSH 隧道能访问 CampusVision C1 服务；未上传图片时可使用本地模拟流程演示，上传图片后的检测失败、无匹配、超时或真实检索失败不会展示模拟命中。

# GKGuard v0.1.30

## Highlights

- Fixed the default multi-face selection modal size: the modal is now shown before calculating the image fit, so the default state and `适应` show the full original image at the largest no-scroll size allowed by the visible frame.
- Expanded desktop modal space for multi-face selection: the modal surface now uses a fixed available height and a wider desktop cap so large-image selection uses more of the window.
- Fixed the result portrait source: the portrait now prefers the selected query-face crop from the target box and falls back to the CampusVision C1 face thumbnail only when needed.
- Fixed query-face crop boundaries: local cropping uses the selected target bbox directly without extra expansion, and very small test images no longer produce empty `data:,` images.
- Synchronized README, API specification, CampusVision C1 / GKGuard C2 integration notes, auto-connection guide, demo script, documentation index, icon notes, and package versions to `v0.1.30`.

## Behavior Notes

- The multi-face modal default state and `适应` use the largest size where both image dimensions remain fully visible without scrolling. Users can still zoom out or zoom in for precise selection.
- The result-side portrait represents the current query target, so it uses the target box selected from the uploaded image. Record thumbnails still prefer CampusVision C1 face thumbnails.
- This release does not modify the remote CampusVision C1 service and does not add real surveillance, face, or personal-information data to the repository.

## Verification

- Passed: `node --check backend/app/static/app.js`.
- Passed: `python -m pytest backend`.
- Passed: `npm run test:e2e`, covering maximum no-scroll multi-face modal fit, `适应` returning to maximum full-image fit after zoom-out, selected query-face result portrait, keyframe target overlays, no-match/timeout recovery, and responsive layout.
- Release flow requires GitHub Actions `Verify` and the Release workflow to pass, followed by checking that the official GitHub Release contains the installer, `.blockmap`, and `latest.yml`.

## Installation

For a first-time installation, download and run `GKGuard-Setup-0.1.30.exe`. Existing installations can update through the top-right `检查更新` entry.

Real CampusVision C1 search still requires campus network, VPN, or an SSH tunnel to the CampusVision C1 service. The mock flow remains available when no image has been uploaded; uploaded-image detection failures, no-match results, timeouts, or real-search failures do not show mock hits.
