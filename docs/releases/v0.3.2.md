# GKGuard v0.3.2

`v0.3.2` 是 `v0.3.1` 之后的人物特征搜索一致性修复版本。版本保留 `v0.3.0` 引入的 GKGuard C2 Workbench / Evidence Desk 信息架构，重点修复 CampusVision C1 人物特征搜索结果中记录切换、目标框标注、候选人物聚合和重复事件详情入口的问题。

## 中文

### 变更摘要

- 修复人物特征搜索结果左侧人物照片：点击不同检索记录时，人物照片同步切换为当前记录对应人物的人脸，不再固定显示第一条记录。
- 为人物特征搜索关键帧补齐目标人脸框和相似度标注，优先使用 CampusVision C1 返回的 `faceBox` / `face_box` / `targetFaceBox` 等字段，行为与人脸以图搜人结果保持一致。
- 候选人物抽屉改为按人物身份聚合：同一人物在多个场景出现时只显示一个候选项，并展示出现次数；选择候选人物后，左侧检索记录只显示该人物对应场景，可恢复显示全部记录。
- 移除结果页重复的 `事件详情` 按钮。事件证据、关键帧、相似度、相机和属性信息统一保留在主详情区，避免抽屉与主屏展示重复内容。
- 同步 README、API 规范、CampusVision C1 / GKGuard C2 集成说明、数据字典、演示脚本、设计说明、支持/安全文档和 AI agent 协作规范。

### 影响范围

- GKGuard C2 前端 / 静态页面
- GKGuard C2 后端 / FastAPI 静态资源版本校验
- CampusVision C1 代理返回字段展示
- 文档 / 发布说明

## English

### Summary

- Fixed the person-attribute result portrait: when the operator clicks a different result record, the left target portrait now follows that record's person face instead of staying pinned to the first record.
- Added target-face boxes and similarity labels to person-attribute keyframes, prioritizing CampusVision C1 fields such as `faceBox`, `face_box`, and `targetFaceBox`, so the behavior matches face-image search results.
- Changed the candidate-person drawer to group by person identity. The same person across multiple scenes appears once with an occurrence count; selecting a candidate filters the left result records to that person's scenes, with a control to restore all records.
- Removed the duplicate `事件详情` button from result pages. Event evidence, keyframes, similarity, camera, and attribute information now remain in the main detail surface to avoid duplicating the same content in a drawer.
- Synchronized README, API specifications, CampusVision C1 / GKGuard C2 integration notes, data dictionary, demo script, design notes, support/security documents, and AI-agent collaboration rules.

- GKGuard C2 backend / FastAPI static asset version checks
