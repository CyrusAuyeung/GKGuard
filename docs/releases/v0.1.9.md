# GKGuard v0.1.9

## 中文

## 重点更新

- 实现打开即用的 CampusVision C1 自动连接体验：安装后打开 GKGuard，会自动启动内置本地 GKGuard C2 后端并探测 CampusVision C1 服务。
- 内置 CampusVision C1 候选地址：`http://10.4.167.122:8000` 和 `http://127.0.0.1:18000`。
- 如果未检测到 CampusVision C1 服务，桌面端会弹出“连接 CampusVision C1 服务”提示，并打开 PowerShell SSH 窗口。
- 用户只需在 PowerShell 中输入服务器密码，GKGuard 不保存、不记录、不直接处理密码。
- SSH 隧道默认使用 `speng@10.4.167.122`，映射 `127.0.0.1:18000 -> 127.0.0.1:8000`。
- GKGuard C2 后端支持从环境变量、配置文件、内置服务器地址和默认隧道地址中自动选择健康的 CampusVision C1 服务。
- 新增 [docs/c1_auto_connection.md](https://github.com/CyrusAuyeung/GKGuard/blob/main/docs/c1_auto_connection.md) 和 [docs/examples/c1-connection.example.json](https://github.com/CyrusAuyeung/GKGuard/blob/main/docs/examples/c1-connection.example.json)，说明自动连接、覆盖配置和 SSH 隧道策略。

## 接口与集成

- `GET /c1/status` 现在返回 `selectedBaseUrl`、`candidateUrls` 和每个候选地址的健康检查状态。
- `/c1/search/person-by-image` 会使用自动选中的 CampusVision C1 地址执行真实以图搜人。
- CampusVision C1 不可达时，前端仍会回退到本地模拟数据，保证演示不会中断。
- 安装版 Electron 会向后端传入 CampusVision C1 配置路径，并默认提供内置候选地址。

## 运行说明

- 下载 `GKGuard-Setup-0.1.9.exe` 安装后，直接打开即可进入 GKGuard。
- 如果 CampusVision C1 可直连，软件会自动使用真实 CampusVision C1 服务。
- 如果 CampusVision C1 只能通过服务器本机访问，软件会提示打开 SSH 窗口；在 PowerShell 中输入服务器密码后即可建立隧道。
- 不在校园网或密码未输入时，软件仍可使用本地模拟回退演示三屏流程。
- 如服务器地址、账号或端口变化，可用 `%APPDATA%\GKGuard\c1-connection.json` 覆盖默认配置；普通使用不需要手动创建该文件。

## 文档更新

- README 已说明免配置连接、SSH 密码窗口和自动 CampusVision C1 探测行为。
- CampusVision C1 / GKGuard C2 集成文档已同步候选地址自动探测顺序。
- 自动连接文档已提供中文默认、English 跳转的完整说明。

# GKGuard v0.1.9

## English

## Highlights

- Added an install-and-open CampusVision C1 auto-connection experience: after installing and opening GKGuard, the app starts the bundled local GKGuard C2 backend and probes CampusVision C1 automatically.
- Built-in CampusVision C1 candidates: `http://10.4.167.122:8000` and `http://127.0.0.1:18000`.
- If CampusVision C1 is not detected, the desktop app shows a “Connect CampusVision C1 service” prompt and opens a PowerShell SSH window.
- The user only enters the server password in PowerShell. GKGuard does not store, log, or directly handle the password.
- The default SSH tunnel uses `speng@10.4.167.122` and maps `127.0.0.1:18000 -> 127.0.0.1:8000`.
- The GKGuard C2 backend can automatically select a healthy CampusVision C1 service from environment variables, config file, built-in server URL, and default tunnel URL.
- Added [docs/c1_auto_connection.md](https://github.com/CyrusAuyeung/GKGuard/blob/main/docs/c1_auto_connection.md) and [docs/examples/c1-connection.example.json](https://github.com/CyrusAuyeung/GKGuard/blob/main/docs/examples/c1-connection.example.json) for auto-connection, override config, and SSH tunnel strategy.

## APIs And Integration

- `GET /c1/status` now returns `selectedBaseUrl`, `candidateUrls`, and health status for each candidate URL.
- `/c1/search/person-by-image` uses the automatically selected CampusVision C1 URL for real image-based person search.
- If CampusVision C1 is unavailable, the frontend still falls back to local mock data so the demo does not stop.
- The packaged Electron app passes the CampusVision C1 config path to the backend and provides built-in candidate URLs by default.

## Runtime Notes

- Download and install `GKGuard-Setup-0.1.9.exe`, then open GKGuard directly.
- If CampusVision C1 is directly reachable, the app uses the real CampusVision C1 service automatically.
- If CampusVision C1 is only reachable from the server itself, the app prompts to open an SSH window; entering the server password in PowerShell establishes the tunnel.
- Outside campus network or without entering the password, the app still demonstrates the three-screen flow through local mock fallback.
- If the server address, account, or ports change, `%APPDATA%\GKGuard\c1-connection.json` can override defaults. Normal use does not require creating this file manually.

## Documentation Updates

- README now explains no-extra-config connection, SSH password window, and automatic CampusVision C1 probing.
- CampusVision C1 / GKGuard C2 integration docs now include the candidate URL probing order.
- Auto-connection docs provide complete Chinese-first bilingual instructions.
- Release notes continue the detailed bilingual format for later maintenance and demos.

## Verification

- Electron main-process syntax: `node --check desktop/main.js` passed.
- Backend tests: `21 passed, 1 warning`.
- `git diff --check` passed.
- Verified Markdown docs keep the Chinese/English top navigation structure.
- Confirmed the worktree was clean before release.

## Installer Assets

This release includes:

- `GKGuard-Setup-0.1.9.exe`: Windows desktop installer.
- `GKGuard-Setup-0.1.9.exe.blockmap`: Electron differential update metadata.
- `latest.yml`: Electron update-channel metadata.

For normal installation, download and run `GKGuard-Setup-0.1.9.exe`.
