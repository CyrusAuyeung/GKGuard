# GKGuard v0.2.0

## 变更摘要

`v0.2.0` 是 CampusVision C1 与 GKGuard C2 的功能级集成版本。CampusVision C1 新增面向 GKGuard C2 的人脸查询大接口、人物特征搜索、人物事件、事件观测、事件关键帧、事件人体图和观测媒体读取；GKGuard C2 新增对应代理层与人物特征搜索界面，使桌面工作台同时支持“上传目标人脸以图搜人”和“按上装颜色、眼镜状态、外观倾向、摄像头、时间范围检索事件”两类演示入口。

本版同时合入 CampusVision C1 服务端稳定性修复和路由保护补齐，保留 Windows 客户端更新能力，并继续生成 Windows、macOS 和 Linux 桌面端发布资产。

## 主要更新

- CampusVision C1 新增 `POST /api/v1/query/face-image`，返回候选人物、人物属性、事件、逐帧匹配和轨迹，供 GKGuard C2 直接展示候选与事件列表。
- CampusVision C1 新增 `POST /api/v1/query/person-attributes`，支持按时间、摄像头、上装颜色、眼镜状态、外观倾向和人物范围检索事件。
- CampusVision C1 新增或补齐 `GET /api/v1/events`、`GET /api/v1/persons/{person_id}/events`、`GET /api/v1/events/{event_id}/observations`。
- CampusVision C1 新增事件和观测媒体读取：`/api/v1/media/event/frame/{event_id}`、`/api/v1/media/event/body/{event_id}`、`/api/v1/media/observation/frame/{observation_id}`、`/api/v1/media/observation/body/{observation_id}`。
- GKGuard C2 新增 `/c1/events`、`/c1/persons/{person_id}/events`、`/c1/events/{event_id}/observations`、`/c1/query/face-image`、`/c1/query/person-attributes` 和扩展版 `/c1/media/{media_path}` 代理。
- GKGuard C2 前端新增 `人脸以图搜人` / `人物特征搜索` 分段入口。人物特征搜索不需要上传图片，支持筛选上装颜色、眼镜状态、外观倾向、摄像头、时间范围、最低分和返回数量。
- 人物特征搜索结果页会展示事件关键帧、人体图或人脸图，并在 `相关信息` 中展示上装颜色、眼镜状态、外观倾向、匹配类型、未满足条件和条件评分。
- `exact` 表示已填写条件全部满足；`partial` 表示相似但部分条件不满足；`unknown` 表示 CampusVision C1 无法判断对应属性，不等同于否定结果。
- 人物特征搜索中显式选择 `unknown` 时，真实 `unknown` 事件会作为满足条件处理；候选扫描优先覆盖最新事件窗口，避免大规模索引只检索到最早事件。
- CampusVision C1 受保护路径补齐到 `/api/v1/...` 查询、事件、实时源和媒体路径，避免新增接口绕过 API key 保护。
- CampusVision C1 失败索引回滚会清理本次尝试新增的人脸记录、人物观测和帧文件，避免重试或人物特征搜索读取脏观测。
- CampusVision C1 查询图大接口复用上传数量和单文件大小限制，超限时返回 413；手动实时源采集缺省记录采集开始时间，保证索引事件可按时间筛选。
- CampusVision C1 人物事件接口支持 `limit` 透传；GKGuard C2 人物特征搜索按 CampusVision C1 实际返回数量生成记录，不再固定截断到 20 条。
- GKGuard C2 人物特征搜索保留 ISO 时间格式，并只在 CampusVision C1 明确返回人体图 URL 时使用人体图，否则回退到人脸图或关键帧。
- CampusVision C1 保留事件装束分组、上装颜色、眼镜状态、外观倾向、人物库、任务报告和评估脚本，供后续真实数据接入和模型评估继续迭代。
- 静态资源版本参数更新为 `v0.2.0-ui`，避免安装版继续复用旧前端资源。

## 行为边界

- 人物特征搜索是可解释筛选与相似事件检索，不是身份确认。低分、`partial` 和 `unknown` 结果必须人工判断。
- 上装颜色、眼镜状态和外观倾向来自 CampusVision C1 模型/profile 输出，不能写成绝对事实。
- CampusVision C1 当前仍以本地服务和当前样本库闭环为主；真实学校监控流接入、长时间真实运行和更大规模索引仍需要后续验证。
- 事件装束分组已有可重映射评估链路，但部分指标仍未达到稳定上线目标；本版不把装束分组能力描述为生产完成。
- 本仓库不提交真实视频、真实查询图片、抽帧图、人脸裁剪图、人体裁剪图、数据库、`.env`、服务器凭据或个人信息。

## 安装与更新

- Windows：下载并运行 `GKGuard-Setup-0.2.0.exe`。已安装旧版时，可使用软件右上角 `检查更新` 入口更新。
- macOS：下载 `GKGuard-macOS-*.dmg` 或 `GKGuard-macOS-*.zip`。当前仍是未签名、未公证的内部测试包。
- Linux：下载 `GKGuard-Linux-*.AppImage` 或 `GKGuard-Linux-*.deb`。
- Release 中应同时包含 `.blockmap`、`latest.yml`、`latest-mac.yml` 和 `latest-linux.yml`。

# GKGuard v0.2.0

## Summary

`v0.2.0` is a feature-level integration release for CampusVision C1 and GKGuard C2. CampusVision C1 adds GKGuard C2-oriented face-query, person-attribute search, person events, event observations, event keyframes, event body crops, and observation media access. GKGuard C2 adds the matching proxy layer and person-attribute search UI, so the desktop workbench now supports both "upload a target face to search people" and "search events by upper color, glasses status, appearance presentation, camera, and time range" demo entries.

This release also includes CampusVision C1 service-stability fixes and completed route protection, keeps Windows client update compatibility, and continues to produce Windows, macOS, and Linux desktop release assets.

## Highlights

- CampusVision C1 adds `POST /api/v1/query/face-image`, returning candidate people, attributes, events, frame-level matches, and trajectory for direct GKGuard C2 presentation.
- CampusVision C1 adds `POST /api/v1/query/person-attributes`, supporting event search by time, camera, upper color, glasses status, appearance presentation, and person scope.
- CampusVision C1 adds or completes `GET /api/v1/events`, `GET /api/v1/persons/{person_id}/events`, and `GET /api/v1/events/{event_id}/observations`.
- CampusVision C1 adds event and observation media access: `/api/v1/media/event/frame/{event_id}`, `/api/v1/media/event/body/{event_id}`, `/api/v1/media/observation/frame/{observation_id}`, and `/api/v1/media/observation/body/{observation_id}`.
- GKGuard C2 adds `/c1/events`, `/c1/persons/{person_id}/events`, `/c1/events/{event_id}/observations`, `/c1/query/face-image`, `/c1/query/person-attributes`, and the extended `/c1/media/{media_path}` proxy.
- The GKGuard C2 frontend adds `人脸以图搜人` / `人物特征搜索` segmented entries. Person-attribute search does not require an uploaded image and supports upper color, glasses status, appearance presentation, camera, time range, minimum score, and result limit filters.
- The person-attribute result page shows event keyframes, body crops, or face crops, and the `相关信息` section shows upper color, glasses status, appearance presentation, match type, failed conditions, and condition scores.
- `exact` means all filled conditions match. `partial` means the event is similar but some conditions failed. `unknown` means CampusVision C1 cannot determine that attribute, not that the attribute is false.
- When person-attribute search explicitly selects `unknown`, events whose actual value is `unknown` satisfy that condition. Candidate scanning prioritizes the newest event window so large indexes do not only evaluate the oldest events.
- CampusVision C1 protected paths now cover `/api/v1/...` query, event, live-source, and media paths so newly added endpoints do not bypass API-key protection.
- Failed CampusVision C1 indexing now cleans up face records, person observations, and frame files created by the current attempt, preventing retries or person-attribute searches from reading dirty observations.
- The CampusVision C1 face-query endpoint reuses upload-count and single-file-size limits and returns 413 when those limits are exceeded. Manual live-source capture stamps the capture start time by default so indexed events can be filtered by time.
- The CampusVision C1 person-events endpoint now honors the proxied `limit`; GKGuard C2 generates records from the actual CampusVision C1 returned result count instead of truncating at 20.
- GKGuard C2 person-attribute search keeps ISO time values and uses event body crops only when CampusVision C1 explicitly returns a body-crop URL, otherwise falling back to face crops or keyframes.
- CampusVision C1 keeps event outfit grouping, upper color, glasses status, appearance presentation, person index, task reports, and evaluation scripts for later real-data integration and model evaluation.
- The static asset parameter is updated to `v0.2.0-ui` so installed apps do not keep using stale frontend assets.

## Behavior Boundary

- Person-attribute search is explainable filtering and similar-event retrieval, not identity confirmation. Low-score, `partial`, and `unknown` results require human judgment.
- Upper color, glasses status, and appearance presentation come from CampusVision C1 model/profile outputs and must not be treated as absolute facts.
- CampusVision C1 currently focuses on the local service and current sample-library loop. Real school surveillance stream integration, long-running real operation, and larger indexes still need later validation.
- Event outfit grouping has a remappable evaluation path, but some metrics are still below stable-production targets. This release does not describe outfit grouping as production-complete.
- The repository does not commit real videos, real query images, extracted frames, face crops, body crops, databases, `.env`, server credentials, or personal information.

## Install And Update

- Windows: download and run `GKGuard-Setup-0.2.0.exe`. Existing installations can update through the top-right `检查更新` entry.
- macOS: download `GKGuard-macOS-*.dmg` or `GKGuard-macOS-*.zip`. The current package is still unsigned and unnotarized for internal testing.
- Linux: download `GKGuard-Linux-*.AppImage` or `GKGuard-Linux-*.deb`.
- The Release should also include `.blockmap`, `latest.yml`, `latest-mac.yml`, and `latest-linux.yml`.
