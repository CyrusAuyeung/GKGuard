# GKGuard v0.3.1

`v0.3.1` 是 `v0.3.0` 工作台重构后的交互与稳定性修复版本。版本保留 GKGuard C2 Workbench / Evidence Desk 信息架构，并修复人脸以图搜人、人物特征搜索、候选人物抽屉、事件详情抽屉、时间输入和结果页响应式布局中的回归问题。

## 变更摘要

- 统一搜索入口和文案：左侧导航保留 `人脸以图搜人` 与 `人物特征搜索`，移除页面中部重复入口，结果页标题同步为 `人脸以图搜人结果` / `人物特征搜索结果`。
- 两个搜索模式的时间输入默认显示用户当天 `00:00` 到 `23:59`，日期输入年份最多四位；当浏览器无法读取 `datetime-local` 光标位置时，不再误拦截月、日或时间数字输入；人物特征搜索只有在用户编辑时间字段后才提交 `time_range`，避免默认显示值隐式过滤历史事件。
- 修复结果页右上角更新按钮与标题/说明重叠，中等宽度下标题区域和按钮自动避让。
- 修复 CampusVision C1 事件详情抽屉：现场图按完整关键帧展示，目标框在抽屉可见后重新定位；`在主视图定位现场图` 会回到主结果画面；字段改为 `标签：值` 格式。
- 修复人物特征搜索结果的大图目标框：GKGuard C2 现在归一化 CampusVision C1 返回的 face/person/body bbox，并在关键帧上标注目标人脸或目标人物。
- 修复候选人物抽屉：优先展示 CampusVision C1 的真实 `candidates`、`people` 和 near misses；缺少真实候选时才使用记录派生候选，避免把同一人物的多条关键帧误当成多个候选人；候选缩略图使用完整自适应展示，不再被方框截断。
- 同步 CampusVision C1 人物特征事件接口的代表观测、face bbox、person/body bbox 到 GKGuard C2 结果模型，并将属性检索记录列表的缩略图优先恢复为人体图，便于复核衣着与人体属性。
- CampusVision C1 人物特征搜索恢复轻量事件扫描，只在返回页确定后补齐代表观测和媒体字段，避免宽泛查询在分页前解析大量 observation JSON。

- `npm run test:e2e`
- `npm audit --audit-level=low`
- `git diff --check`

## 安全与数据

- 未提交真实监控视频、真实查询图片、人脸裁剪图、抽帧图或个人身份信息。
- 未提交 `.env`、密码、token、SSH 私钥或服务器凭据。
- 本版仍只通过 GKGuard C2 后端代理访问 CampusVision C1，前端不直接访问 CampusVision C1。

# GKGuard v0.3.1

`v0.3.1` is an interaction and stability patch after the `v0.3.0` workbench redesign. It keeps the GKGuard C2 Workbench / Evidence Desk information architecture and fixes regressions in face image search, person-attribute search, the candidate-person drawer, the event-detail drawer, time inputs, and responsive result-page layout.

## Change Summary

- Unified search entries and wording: the left navigation keeps `人脸以图搜人` and `人物特征搜索`, the duplicated middle-page entry is removed, and result headings now use `人脸以图搜人结果` / `人物特征搜索结果`.
- Both search modes display the user's current day from `00:00` to `23:59` by default and limit years to four digits; when the browser cannot expose the `datetime-local` cursor position, month/day/time digit input is no longer blocked incorrectly; person-attribute search submits `time_range` only after the user edits a time field, so displayed defaults do not implicitly filter historical events.
- Fixed result-page update-button overlap with the heading/subtitle; medium-width layouts now let the heading area and button avoid each other.
- Fixed the CampusVision C1 event-detail drawer: scene images render as full keyframes, target overlays are repositioned after the drawer is visible, `在主视图定位现场图` returns to the main result view, and fields now use `label: value` formatting.
- Fixed target overlays for person-attribute search results: GKGuard C2 now normalizes CampusVision C1 face/person/body bboxes and marks the target face or target person on keyframes.
- Fixed the candidate-person drawer: real CampusVision C1 `candidates`, `people`, and near misses are shown first; record-derived candidates are used only when no real candidates are available, avoiding duplicate candidate people from multiple keyframes for the same person; candidate thumbnails use full responsive display instead of being clipped.
- Synchronized CampusVision C1 representative observations, face bboxes, and person/body bboxes into the GKGuard C2 result model, and restored body-crop thumbnails as the preferred record-list thumbnail for attribute-search records so clothing/body attributes remain reviewable.
- CampusVision C1 person-attribute search now keeps event scanning lightweight and hydrates representative observations and media fields only after the returned page is known, avoiding broad-query observation JSON parsing before pagination.

## Security And Data

- No real surveillance videos, real query images, face crops, extracted frames, or personal identity data are committed.
- No `.env`, passwords, tokens, SSH private keys, or server credentials are committed.
- This version still accesses CampusVision C1 only through the GKGuard C2 backend proxy; the frontend does not call CampusVision C1 directly.
