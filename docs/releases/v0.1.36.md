# GKGuard v0.1.36

## 变更摘要

`v0.1.36` 是 `v0.1.35` 安全加固后的跟进版本，重点修复 CampusVision C1 接入链路中的身份确认、接口保护、异常回滚和 review 流程问题。Electron 内嵌 SSH 隧道改为自动比对固定主机密钥指纹，CampusVision C1 服务端补齐视频、人物库、记录和媒体读取接口的 API key 保护，同时修复查询图超限、失败索引清理和图片画廊鉴权兼容问题。

## 主要更新

- Electron 内嵌 SSH 隧道在连接 CampusVision C1 前自动比对固定主机密钥指纹；不再依赖每次人工确认，也不会在缺少可信指纹时继续连接。
- GKGuard C2 访问 CampusVision C1 时优先读取 `CAMPUSVISION_API_KEY`，再回退到 `C1_API_KEY`，与 CampusVision C1 服务端配置口径保持一致。
- CampusVision C1 API key 保护补齐到视频、人物库、记录、检索、关键帧和人脸裁剪媒体接口，避免只保护部分路径。
- CampusVision C1 人物库画廊改用内联人脸裁剪图，保证媒体读取接口受保护后浏览器页面仍能正常展示。
- 查询图解码后如果超过像素上限，会返回 413 并清理上传临时文件，不再被误判成“未检测到人脸”。
- 视频索引失败时只回滚本次索引创建的人脸记录和关键帧文件，避免误删同一视频此前已经存在的索引结果。
- `AGENTS.md`、`CONTRIBUTING.md` 和 PR 模板补充 AI agent review 双语要求；PR、commit、diff 或工作区审查输出需要同时包含中文和 English。

## 安装与更新

- Windows：下载并运行 `GKGuard-Setup-0.1.36.exe`。已安装旧版时，可使用软件右上角 `检查更新` 入口更新。
- macOS：下载 `GKGuard-macOS-*.dmg` 或 `GKGuard-macOS-*.zip`。当前仍是未签名、未公证的内部测试包。
- Linux：下载 `GKGuard-Linux-*.AppImage` 或 `GKGuard-Linux-*.deb`。
- Release 中应同时包含 `.blockmap`、`latest.yml`、`latest-mac.yml` 和 `latest-linux.yml`。

## 运维注意

- 如果 CampusVision C1 服务器、端口或 SSH 主机密钥发生变化，需要同步更新 Electron 默认连接配置或 `C1_SSH_HOST_FINGERPRINT`。
- CampusVision C1 开启 API key 后，GKGuard C2 应配置 `CAMPUSVISION_API_KEY`；如需兼容旧环境，可继续使用 `C1_API_KEY`。
- 人物库画廊页面不会再直接引用受保护的人脸裁剪媒体 URL；外部调用这些媒体接口仍需携带 `X-CampusVision-API-Key`。
- 远端 CampusVision C1 服务更新后，应重启服务并确认 `/health`、`/openapi.json` 和 `/api/v1/search/person-by-image` 可用。

# GKGuard v0.1.36

## Summary

`v0.1.36` is a follow-up security-hardening release after `v0.1.35`, focused on CampusVision C1 identity verification, route protection, error rollback, and review-process consistency. The Electron embedded SSH tunnel now automatically compares a pinned host-key fingerprint, CampusVision C1 adds API-key protection for video, person, record, and media-read endpoints, and query-image limits, failed indexing rollback, and gallery authentication compatibility are fixed.

## Highlights

- The Electron embedded SSH tunnel automatically compares the pinned CampusVision C1 host-key fingerprint before connecting. It no longer depends on repeated manual confirmation and refuses to continue when no trusted fingerprint is configured.
- GKGuard C2 now reads `CAMPUSVISION_API_KEY` before falling back to `C1_API_KEY`, matching the CampusVision C1 service configuration.
- CampusVision C1 API-key protection now covers video, person, record, search, keyframe, and face-crop media endpoints instead of only a partial path set.
- The CampusVision C1 person gallery now uses inline face crops so browser pages still render correctly after protected media reads.
- Query images that exceed the decoded pixel limit now return 413 and clean up uploaded temporary files instead of being misreported as no-face detections.
- Failed video indexing only rolls back face records and keyframe files created by the current indexing run, avoiding accidental deletion of existing results for the same video.
- `AGENTS.md`, `CONTRIBUTING.md`, and the PR template now require bilingual AI-agent review output. Reviews of PRs, commits, diffs, or working-tree changes should include both Chinese and English.

## Install And Update

- Windows: download and run `GKGuard-Setup-0.1.36.exe`. Existing installations can update through the top-right `检查更新` entry.
- macOS: download `GKGuard-macOS-*.dmg` or `GKGuard-macOS-*.zip`. The current package is still unsigned and unnotarized for internal testing.
- Linux: download `GKGuard-Linux-*.AppImage` or `GKGuard-Linux-*.deb`.
- The Release should also include `.blockmap`, `latest.yml`, `latest-mac.yml`, and `latest-linux.yml`.

## Operations Notes

- If the CampusVision C1 server, port, or SSH host key changes, update the Electron default connection configuration or `C1_SSH_HOST_FINGERPRINT`.
- When CampusVision C1 API-key protection is enabled, GKGuard C2 should set `CAMPUSVISION_API_KEY`. `C1_API_KEY` remains available for legacy environments.
- The CampusVision C1 person gallery no longer directly references protected face-crop media URLs. External callers still need `X-CampusVision-API-Key` for those media endpoints.
- After updating the remote CampusVision C1 service, restart it and verify `/health`, `/openapi.json`, and `/api/v1/search/person-by-image`.
