# GKGuard v0.1.27

## 重点更新

- 提升 CampusVision C1 查询图人脸检测稳定性：查询图会经过 EXIF 转正、RGB 标准化、透明通道处理、贴边/大脸补边重试和小图放大重试，再把检测框映射回原图坐标。
- 改进多人查询图选择：多张有效候选人脸时，GKGuard C2 会在上传页显示候选框并自动打开放大选择弹窗，用户可放大原图后确认目标人脸。
- 调整低置信候选策略：检测置信度低于 `0.65` 但不低于 `0.45` 的查询人脸会以低置信样式显示并仍可选择，低于 `0.45` 的候选不作为可选目标；检测置信度仍不等同于人物匹配相似度。
- 修复结果关键帧相似度遮挡：目标人脸框继续按实际图片内容区域定位，相似度标签移动到框外，避免遮挡目标人脸。
- 修复检索卡死状态：查询图检测、真实检索、无匹配和超时都会回到可继续操作的上传页，不再长时间停留在“检索中”。
- 修复 Electron 页面缓存残留：桌面端继续清理 renderer cache，并使用 `asset=v0.1.27-ui` 页面参数加载最新前端资源。
- 新增 CampusVision C1 远端半自动部署脚本和说明：`scripts/deploy_c1_remote.ps1` 可通过 SSH alias 更新远端 `main`、安装依赖、重启 uvicorn 并检查 `/health` 与 `query-faces`。
- 同步 README、API 规范、CampusVision C1 / GKGuard C2 集成说明、自动连接说明、远端部署说明、数据字典、演示脚本、CampusVision C1 子服务文档、文档索引和版本号到 `v0.1.27`。

## 行为说明

- 单人查询图会自动选择并检索；多人查询图需要在放大弹窗中确认目标。
- 查询图检测置信度只表示“是否像一张可用人脸”，结果页相似度才表示与 CampusVision C1 人物库/关键帧的匹配程度。
- 上传图片后的查询图检测失败、无匹配、真实检索失败或超时不会回退到本地模拟命中。
- 本版本不向仓库加入真实监控、人脸或个人信息数据；本地测试图片位于被 git 忽略的 `local-test-data/`。

# GKGuard v0.1.27

## Highlights

- Improved CampusVision C1 query-face detection reliability: query images are normalized for EXIF orientation, RGB channels, and alpha, then retried with padding for tight or large faces and with small-image upscaling before boxes are mapped back to original-image coordinates.
- Improved multi-face query selection: when multiple effective candidate faces are detected, GKGuard C2 shows candidate boxes on the upload screen and opens an enlarged selection modal so the user can zoom the original image and confirm the target face.
- Adjusted low-confidence candidate behavior: query faces below `0.65` but at least `0.45` are shown with a low-confidence style and remain selectable, while candidates below `0.45` are not exposed as targets. Detection confidence is still not person-match similarity.
- Fixed result-keyframe similarity overlap: target boxes still use the rendered image content area, and similarity labels now sit outside the box so they do not cover the target face.
- Fixed stuck search states: query-face detection, real search, no-match results, and timeouts return to an actionable upload screen instead of leaving the UI in `检索中`.
- Fixed Electron stale page resources: the desktop app continues to clear the renderer cache and loads frontend resources with `asset=v0.1.27-ui`.
- Added a semi-automatic CampusVision C1 remote deployment script and guide: `scripts/deploy_c1_remote.ps1` can use an SSH alias to update remote `main`, install dependencies, restart uvicorn, and verify `/health` plus `query-faces`.
- Synchronized README, API specification, CampusVision C1 / GKGuard C2 integration notes, auto-connection guide, remote deployment guide, data dictionary, demo script, CampusVision C1 service docs, documentation index, and package versions to `v0.1.27`.

## Behavior Notes

- Single-face query images auto-select and search; multi-face query images require confirming the target in the enlarged modal.
- Query-face detection confidence only describes whether the detected region is a usable face. Result similarity describes the match against CampusVision C1 person/keyframe records.
- Uploaded-image query-face failures, no-match results, real-search failures, and timeouts do not fall back to local mock hits.
- This release does not add real surveillance, face, or personal-information data to the repository; local test images stay under git-ignored `local-test-data/`.

- Passed: `node --check backend/app/static/app.js`, `node --check desktop/main.js`, and `node --check desktop/preload.js`.
- Passed: `python -m py_compile backend/desktop_server.py`.

- Passed: `npm run test:e2e`, with 18 Chromium cases covering local mock flow, CampusVision C1 media results, enlarged multi-face selection, low-confidence candidates, no-match recovery, search timeout recovery, query-face detection failure blocking, outside-box similarity labels, and no page-level horizontal overflow at `1280x720`, `680x640`, and `390x720`.
- Passed: in-app Browser rendering validation at `http://127.0.0.1:8002/demo?desktop=1&asset=v0.1.27-ui`; title was correct, the main screen was non-empty, there was no page-level horizontal overflow, and console warning/error count was 0.

- Passed: `npm run dist`, producing `GKGuard-Setup-0.1.27.exe`, `GKGuard-Setup-0.1.27.exe.blockmap`, and `latest.yml`.
- Release flow requires GitHub Actions `Verify` and the Release workflow to pass, followed by checking that the official GitHub Release contains the installer, `.blockmap`, and `latest.yml`.

## Installation

For a first-time installation, download and run `GKGuard-Setup-0.1.27.exe`. Existing installations can update through the top-right `检查更新` entry.

Real CampusVision C1 search still requires campus network, VPN, or an SSH tunnel to the CampusVision C1 service. The mock flow remains available when no image has been uploaded; uploaded-image detection failures, no-match results, timeouts, or real-search failures do not show mock hits.
