# GKGuard v0.1.28

## 重点更新

- 修复 CampusVision C1 远端半自动部署脚本在 Windows PowerShell 下通过 SSH 发送 bash 脚本时携带 CRLF 的问题，避免远端出现 `set: pipefail\r: invalid option name`。
- 部署脚本改为用 .NET Process 向 SSH 标准输入写入已归一化为 LF 的脚本内容，兼容当前 Windows PowerShell 环境。
- 修正 CampusVision C1 查询图预处理测试中的坐标期望，使远端 conda 环境中的测试结果与当前补边/缩放映射算法一致。
- 同步 README、API 规范、CampusVision C1 / GKGuard C2 集成说明、自动连接说明、演示脚本、文档索引、图标说明和版本号到 `v0.1.28`。

## 行为说明

- 本版本是 `v0.1.27` 的发布流程热修版本，桌面端人脸以图搜人交互、多人放大弹窗选人、低置信候选、结果关键帧框外相似度和无匹配/超时恢复逻辑保持不变。
- CampusVision C1 远端部署仍通过 SSH alias 执行；默认 alias 为 `gkguard-c1`，远端服务继续运行在 `127.0.0.1:8000`。
- 本版本不向仓库加入真实监控、人脸或个人信息数据；本地测试图片继续位于被 git 忽略的 `local-test-data/`。

## 验证

- 通过：`git diff --check`。
- 通过：`python -m pytest services/campusvision-c1/tests`，本地缺少 OpenCV 时按测试级跳过。
- 通过：`scripts/deploy_c1_remote.ps1` 远端部署脚本，CampusVision C1 在远端完成 `git pull`、依赖检查、uvicorn 重启、`/health` 检查和 `query-faces` OpenAPI 检查。
- 通过：远端 CampusVision C1 conda 环境执行 `python -m pytest tests`。
- 发布流程要求 GitHub Actions `Verify` 和 Release workflow 通过，并在正式 GitHub Release 中核对安装包、`.blockmap` 和 `latest.yml` 齐全。

## 安装

普通首次安装下载并运行 `GKGuard-Setup-0.1.28.exe`。已安装旧版时，可使用软件右上角 `检查更新` 入口更新。

真实 CampusVision C1 检索仍需要校园网、VPN 或 SSH 隧道能访问 CampusVision C1 服务；未上传图片时可使用本地模拟流程演示，上传图片后的检测失败、无匹配、超时或真实检索失败不会展示模拟命中。

# GKGuard v0.1.28

## Highlights

- Fixed CRLF handling in the semi-automatic CampusVision C1 remote deployment script when Windows PowerShell sends a bash script through SSH, avoiding `set: pipefail\r: invalid option name` on the remote host.
- The deployment script now writes LF-normalized script content to SSH standard input through a .NET Process, which works with the current Windows PowerShell environment.
- Corrected the CampusVision C1 query-image preprocessing test expectation so the remote conda environment matches the current padding and scaling coordinate-mapping algorithm.
- Synchronized README, API specification, CampusVision C1 / GKGuard C2 integration notes, auto-connection guide, demo script, documentation index, icon notes, and package versions to `v0.1.28`.

## Behavior Notes

- This is a release-process hotfix for `v0.1.27`; the desktop face-search interaction, enlarged multi-face selection modal, low-confidence candidates, outside-box keyframe similarity labels, and no-match/timeout recovery behavior are unchanged.
- CampusVision C1 remote deployment still runs through an SSH alias. The default alias is `gkguard-c1`, and the remote service continues to run on `127.0.0.1:8000`.
- This release does not add real surveillance, face, or personal-information data to the repository; local test images remain under git-ignored `local-test-data/`.

## Verification

- Passed: `git diff --check`.
- Passed: `python -m pytest services/campusvision-c1/tests`; tests skip locally when OpenCV is unavailable.
- Passed: `scripts/deploy_c1_remote.ps1` remote deployment, including remote `git pull`, dependency check, uvicorn restart, `/health` check, and `query-faces` OpenAPI check.
- Passed: `python -m pytest tests` in the remote CampusVision C1 conda environment.
- Release flow requires GitHub Actions `Verify` and the Release workflow to pass, followed by checking that the official GitHub Release contains the installer, `.blockmap`, and `latest.yml`.

## Installation

For a first-time installation, download and run `GKGuard-Setup-0.1.28.exe`. Existing installations can update through the top-right `检查更新` entry.

Real CampusVision C1 search still requires campus network, VPN, or an SSH tunnel to the CampusVision C1 service. The mock flow remains available when no image has been uploaded; uploaded-image detection failures, no-match results, timeouts, or real-search failures do not show mock hits.
