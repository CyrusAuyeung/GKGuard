# GKGuard v0.2.2

## 变更摘要

`v0.2.2` 是 `v0.2.1` 之后的 review 后续修复版本。本版不新增主要产品入口，重点收敛 CampusVision C1 查询图候选接口参数位置、GKGuard C2 对 CampusVision C1 4xx 校验错误和 5xx 服务错误的处理方式、CampusVision C1 输入校验失败的桌面端重连边界，以及人物路线图中重复映射点和未映射记录的高亮行为。

## 主要更新

- CampusVision C1 `POST /api/v1/query/face-image` 保持查询图片使用 multipart `files` 字段，查询控制项统一为 Query 参数，包括 `top_k`、`min_score`、`max_gap_sec`、候选开关、事件/匹配数量、摄像头和时间筛选。
- GKGuard C2 调用 CampusVision C1 查询图候选接口时同步使用 Query 参数，避免代理层与 CampusVision C1 原生接口参数位置不一致。
- GKGuard C2 后端保留 CampusVision C1 返回的 4xx 校验错误详情；不可解码查询图、超限查询图等用户输入问题会向前端返回对应错误码和中文提示，不再被统一包装成连接不可用。CampusVision C1 5xx 服务端错误不会透传内部详情，只返回结构化的服务不可用状态。
- GKGuard C2 前端遇到 CampusVision C1 4xx 输入或校验错误时停留在当前可恢复页面，不触发桌面端 CampusVision C1 SSH 重连流程，避免把坏图片、无效检索条件或超限请求误判为服务连接故障。
- 路线图重复映射到同一结果记录的多个路线点只高亮当前选中的路线点；未映射到结果记录的尾部记录不会错误高亮最后一个路线点。
- 补充后端和 Playwright 回归测试，覆盖非重试 4xx 错误透传、CampusVision C1 5xx 详情脱敏、图片检索和人物特征检索 4xx 校验失败不重连、重复映射路线点唯一高亮和未映射记录不误高亮路线点。
- README、文档索引、SECURITY、SUPPORT、API 规范、CampusVision C1 / GKGuard C2 集成说明、自动连接说明、演示脚本、CampusVision C1 文档和图标说明同步为 `v0.2.2` 当前状态。

## 行为边界

- 本版延续 `v0.2.0` / `v0.2.1` 的功能边界：人物特征检索是可解释筛选与相似事件检索，不是身份确认。
- 查询图不可解码、超限或人物特征条件校验失败属于输入校验失败，不表示 CampusVision C1 服务不可用；只有连接失败、健康检查失败或脱敏后的服务端 5xx 才进入连接恢复或重试路径。
- 上装颜色、眼镜状态和外观倾向仍来自 CampusVision C1 模型/profile 输出，不能写成绝对事实。
- 本仓库不提交真实视频、真实查询图片、抽帧图、人脸裁剪图、人体裁剪图、数据库、`.env`、服务器凭据或个人信息。

## 安装与更新

- macOS：下载 `GKGuard-macOS-*.dmg` 或 `GKGuard-macOS-*.zip`。当前仍是未签名、未公证的内部测试包。

# GKGuard v0.2.2

## Summary

`v0.2.2` is a review-follow-up patch after `v0.2.1`. It does not add major product entries. It focuses on CampusVision C1 query-image candidate parameter placement, GKGuard C2 handling of CampusVision C1 4xx validation errors and 5xx service errors, desktop reconnection boundaries for CampusVision C1 input-validation failures, and route-map highlighting for duplicate-mapped points and unmapped result records.

## Highlights

- CampusVision C1 `POST /api/v1/query/face-image` keeps query images in the multipart `files` field, while control fields are consistently Query parameters, including `top_k`, `min_score`, `max_gap_sec`, candidate flags, event/match limits, camera filters, and time filters.
- GKGuard C2 now calls the CampusVision C1 query-image candidate endpoint with Query parameters as well, avoiding parameter-placement drift between the proxy layer and the native CampusVision C1 API.
- The GKGuard C2 backend preserves CampusVision C1 4xx validation-error details. Undecodable or over-limit query images return the corresponding error code and Chinese UI message instead of being collapsed into a generic connection-unavailable error. CampusVision C1 5xx service errors do not expose internal details and return a structured service-unavailable state instead.
- When the GKGuard C2 frontend receives a CampusVision C1 4xx input or validation error, it stays on the current recoverable screen and does not trigger the desktop CampusVision C1 SSH reconnection flow, so bad images, invalid query conditions, or over-limit requests are not misclassified as service-connection failures.
- Route points that map to the same result record highlight only the currently selected route point. Result records without mapped route points no longer incorrectly highlight the final route point.
- Backend and Playwright regression coverage now includes non-retryable 4xx propagation, CampusVision C1 5xx detail sanitization, image-search and person-attribute-search 4xx validation failures without reconnection, single active highlighting for duplicate-mapped route points, and unmapped records not highlighting route points.
- README, docs index, SECURITY, SUPPORT, API specifications, CampusVision C1 / GKGuard C2 integration notes, auto-connection notes, demo script, CampusVision C1 docs, and icon notes now describe `v0.2.2` as the current state.

## Behavior Boundary

- This release keeps the `v0.2.0` / `v0.2.1` functional boundary: person-attribute search is explainable filtering and similar-event retrieval, not identity confirmation.
- Undecodable or over-limit query images and person-attribute condition validation failures are input-validation failures, not CampusVision C1 service availability failures. Only connection failures, health-check failures, or sanitized server-side 5xx responses enter recovery or retry paths.
- Upper color, glasses status, and appearance presentation still come from CampusVision C1 model/profile outputs and must not be treated as absolute facts.
- The repository does not commit real videos, real query images, extracted frames, face crops, body crops, databases, `.env`, server credentials, or personal information.

## Install And Update

- Windows: download and run `GKGuard-Setup-0.2.2.exe`. Existing installations can update through the top-right `检查更新` entry.
- macOS: download `GKGuard-macOS-*.dmg` or `GKGuard-macOS-*.zip`. The current package is still unsigned and unnotarized for internal testing.
- Linux: download `GKGuard-Linux-*.AppImage` or `GKGuard-Linux-*.deb`.
- The Release should also include `.blockmap`, `latest.yml`, `latest-mac.yml`, and `latest-linux.yml`.
