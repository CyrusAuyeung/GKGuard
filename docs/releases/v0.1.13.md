# GKGuard v0.1.13

## 中文

## 重点更新

- 修复校园网环境下 CampusVision C1 直连可达但真实检索返回 HTTP 503 时，桌面端没有提示输入服务器密码、前端直接回退本地模拟的问题。
- 安装版默认优先检查本机 SSH 隧道 `http://127.0.0.1:18000`，再尝试校园网直连地址 `http://10.4.167.122:8000`。
- 前端在 CampusVision C1 检索失败时会调用桌面端连接能力，打开 PowerShell SSH 登录窗口；用户输入服务器密码后自动重试一次真实检索。
- GKGuard C2 适配器遇到 CampusVision C1 502/503/504 会尝试下一个候选 CampusVision C1 地址，降低单个候选服务异常导致静默回退的概率。
- README、API 规范、CampusVision C1 / GKGuard C2 集成说明、自动连接说明、演示脚本、安全支持范围和发布说明已同步到 `v0.1.13` 当前状态。

## 运行说明

- 旧版 `v0.1.12` 可以点击软件右上角 `检查更新`，发现 `v0.1.13` 后再次点击下载新安装包。
- 首次打开或真实检索遇到 CampusVision C1 503 时，选择“输入密码连接 CampusVision C1 服务”，在 PowerShell 窗口输入服务器密码并保持窗口打开。
- GKGuard 不保存、不读取、不记录 SSH 密码。

## 验证情况

- 后端、静态资源、CampusVision C1 retry 和桌面 bridge 回归测试：`24 passed, 1 warning`。
- Electron 主进程和 preload 脚本语法检查通过。
- 浏览器模拟验证通过：第一次 CampusVision C1 检索返回 503 后调用桌面端 `connectC1`，随后自动重试并显示 `CampusVision C1` 结果。
- `npm ci` 通过，`npm audit --audit-level=low` 确认 0 vulnerabilities。
- PyInstaller 后端包构建通过，本地 Windows 安装包构建 `npm run dist` 通过。

## 安装包

本次 GitHub Release 下方包含：

- `GKGuard-Setup-0.1.13.exe`：Windows 桌面安装包。
- `GKGuard-Setup-0.1.13.exe.blockmap`：Electron 更新差分元数据。
- `latest.yml`：Electron 更新通道元数据。

普通安装只需要下载并运行 `GKGuard-Setup-0.1.13.exe`。

# GKGuard v0.1.13

## English

## Highlights

- Fixed the campus-network case where direct CampusVision C1 was reachable but real search returned HTTP 503, while the desktop app did not ask for the server password and the frontend immediately fell back to local mock data.
- The packaged app now prefers the local SSH tunnel `http://127.0.0.1:18000` before trying the campus-network direct URL `http://10.4.167.122:8000`.
- When CampusVision C1 search fails, the frontend calls the desktop connection bridge, opens the PowerShell SSH login window, and retries real search once after the user enters the server password.
- The GKGuard C2 adapter retries the next candidate CampusVision C1 URL on CampusVision C1 502/503/504 to reduce silent fallback caused by one unhealthy candidate.
- Updated README, API specification, CampusVision C1 / GKGuard C2 integration notes, auto-connection guide, demo script, supported security version, and Release notes for the current `v0.1.13` state.

## Runtime Notes

- Existing `v0.1.12` users can click the top-right `检查更新`; when `v0.1.13` is found, click again to download the new installer.
- On first open or when real search hits CampusVision C1 503, choose “输入密码连接 CampusVision C1 服务”, enter the server password in the PowerShell window, and keep that window open.
- GKGuard does not store, read, or log the SSH password.

## Verification

- Backend, static asset, CampusVision C1 retry, and desktop bridge regression tests: `24 passed, 1 warning`.
- Electron main-process and preload syntax checks passed.
- Browser simulation passed: the first CampusVision C1 search returned 503, the frontend called desktop `connectC1`, then automatically retried and displayed `CampusVision C1` results.
- `npm ci` passed, and `npm audit --audit-level=low` confirmed 0 vulnerabilities.
- PyInstaller backend bundling passed, and the local Windows installer build `npm run dist` passed.

## Installer Assets

This release includes:

- `GKGuard-Setup-0.1.13.exe`: Windows desktop installer.
- `GKGuard-Setup-0.1.13.exe.blockmap`: Electron differential update metadata.
- `latest.yml`: Electron update-channel metadata.

For normal installation, download and run `GKGuard-Setup-0.1.13.exe`.
