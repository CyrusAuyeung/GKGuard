# GKGuard v0.1.26

## 重点更新

- 修复 Electron 主进程隧道异常弹窗：CampusVision C1 SSH 隧道连接被远端重置时，桌面端会关闭对应 socket 并记录警告，不再弹出 `read ECONNRESET` JavaScript 错误。
- 修复安装更新后的旧页面缓存问题：桌面端加载演示页前会清理 Electron renderer cache，并使用 `asset=v0.1.26-ui` 页面参数，避免安装新版后继续显示旧 HTML/CSS/JS。
- 修复查询图人脸框显示：查询图和结果关键帧的人脸框不再使用固定最小尺寸；存在高置信候选时，低于 `0.65` 的低置信检测框会被隐藏，减少多人图误检框堆叠。
- 修复结果关键帧目标框定位：GKGuard C2 前端按实际渲染后的图片内容区域计算目标框，兼容像素、归一化和百分比坐标，避免目标框落在左上角或黑边区域。
- 修复 CampusVision C1 无匹配结果状态：当真实检索返回空 `records[]` 或无匹配 warning 时，页面会停留在可继续操作的上传页并显示中文提示，不再卡在“检索中”或进入本地模拟结果。
- 修复 CampusVision C1 远端依赖漂移：`requirements.txt` 和 `environment-from-scratch.yml` 将 `numpy` 限制为 `<2`、`opencv-python` 限制为 `<4.13`，并给 `.env.example` 中包含空格的 `APP_NAME` 加引号。
- 同步 README、API 规范、CampusVision C1 / GKGuard C2 集成说明、自动连接说明、数据字典、演示脚本、CampusVision C1 子服务文档、文档索引、Issue 模板和版本号到 `v0.1.26`。

## 行为说明

- 查询图只有一张有效候选人脸时仍会自动检索；有多张有效候选人脸时需要在原图上选择目标。
- 检测置信度只表示查询图人脸检测质量；结果页相似度仍表示目标人脸与 CampusVision C1 人物库/关键帧的匹配分数。
- CampusVision C1 无匹配、查询图检测失败或真实检索失败时，上传图流程不会回退到本地模拟命中。
- 本版本不向仓库加入真实监控、人脸或个人信息数据。

## 验证

- 通过：`node --check backend/app/static/app.js`、`node --check desktop/main.js`、`node --check desktop/preload.js`。
- 通过：`python -m py_compile backend/desktop_server.py`。
- 通过：`python -m pytest backend`，`27 passed, 1 warning`。
- 通过：`npm audit --audit-level=low`，`found 0 vulnerabilities`。
- 通过：`npm run test:e2e`，15 个 Chromium 用例覆盖 `1280x720`、`680x640` 和 `390x720` 下的本地模拟、CampusVision C1 媒体结果、多人框选、低置信查询框过滤、无匹配状态恢复和查询图检测失败拦截。
- 通过：in-app Browser 渲染验证，`http://127.0.0.1:8002/demo?desktop=1&asset=v0.1.26-ui` 标题正确、主界面非空、无页面级横向溢出，console warning/error 为 0。
- 通过：`npm run dist`，已生成 `GKGuard-Setup-0.1.26.exe`、`GKGuard-Setup-0.1.26.exe.blockmap` 和 `latest.yml`。
- 发布流程要求 GitHub Actions `Verify` 和 Release workflow 通过，并在正式 GitHub Release 中核对安装包、`.blockmap` 和 `latest.yml` 齐全。

## 安装

普通首次安装下载并运行 `GKGuard-Setup-0.1.26.exe`。已安装旧版时，可使用软件右上角 `检查更新` 入口更新。

真实 CampusVision C1 检索仍需要校园网、VPN 或 SSH 隧道能访问 CampusVision C1 服务；未上传图片时可使用本地模拟流程演示，上传图片后的检测失败、无匹配或真实检索失败不会展示模拟命中。

# GKGuard v0.1.26

## Highlights

- Fixed Electron main-process tunnel error dialogs: when the CampusVision C1 SSH tunnel connection is reset by the remote side, the desktop app closes the socket and logs a warning instead of showing a `read ECONNRESET` JavaScript error.
- Fixed stale page caching after installed updates: the desktop app clears the Electron renderer cache before loading the demo page and uses `asset=v0.1.26-ui` so the app does not keep old HTML/CSS/JS after an upgrade.
- Fixed query-face box rendering: query-image and result-keyframe face boxes no longer use a forced minimum size. When higher-confidence candidates exist, boxes below `0.65` are hidden to reduce false-positive overlap in multi-face images.
- Fixed result-keyframe target-box positioning: the GKGuard C2 frontend positions target boxes against the actual rendered image content, accepts pixel, normalized, and percentage coordinates, and avoids top-left or letterbox placement.
- Fixed CampusVision C1 no-match handling: when real search returns an empty `records[]` result or no-match warning, the UI stays on the actionable upload screen with a Chinese warning instead of staying in `检索中` or entering local mock results.
- Fixed CampusVision C1 remote dependency drift: `requirements.txt` and `environment-from-scratch.yml` now keep `numpy<2` and `opencv-python<4.13`, and `.env.example` quotes the space-containing `APP_NAME`.
- Synchronized README, API specification, CampusVision C1 / GKGuard C2 integration notes, auto-connection guide, data dictionary, demo script, CampusVision C1 service docs, documentation index, Issue templates, and package versions to `v0.1.26`.

## Behavior Notes

- Query images with one effective candidate face still auto-search; query images with multiple effective candidate faces require selecting the target on the original image.
- Detection confidence describes query-face detection quality only; result similarity still describes the match between the target face and CampusVision C1 person/keyframe records.
- CampusVision C1 no-match results, query-face detection failures, and real-search failures do not fall back to local mock hits after an image has been uploaded.
- This release does not add real surveillance, face, or personal-information data to the repository.

## Verification

- Passed: `node --check backend/app/static/app.js`, `node --check desktop/main.js`, and `node --check desktop/preload.js`.
- Passed: `python -m py_compile backend/desktop_server.py`.
- Passed: `python -m pytest backend`, `27 passed, 1 warning`.
- Passed: `npm audit --audit-level=low`, `found 0 vulnerabilities`.
- Passed: `npm run test:e2e`, with 15 Chromium cases covering local mock flow, CampusVision C1 media results, multi-face selection, low-confidence query-box filtering, no-match recovery, and query-face detection failure blocking at `1280x720`, `680x640`, and `390x720`.
- Passed: in-app Browser rendering validation at `http://127.0.0.1:8002/demo?desktop=1&asset=v0.1.26-ui`; title was correct, the main screen was non-empty, there was no page-level horizontal overflow, and console warning/error count was 0.
- Passed: `npm run dist`, producing `GKGuard-Setup-0.1.26.exe`, `GKGuard-Setup-0.1.26.exe.blockmap`, and `latest.yml`.
- Release flow requires GitHub Actions `Verify` and the Release workflow to pass, followed by checking that the official GitHub Release contains the installer, `.blockmap`, and `latest.yml`.

## Installation

For a first-time installation, download and run `GKGuard-Setup-0.1.26.exe`. Existing installations can update through the top-right `检查更新` entry.

Real CampusVision C1 search still requires campus network, VPN, or an SSH tunnel to the CampusVision C1 service. The mock flow remains available when no image has been uploaded; uploaded-image detection failures, no-match results, or real-search failures do not show mock hits.
