# GKGuard v0.1.11

## 中文

## 重点更新

- 优化人脸检索结果页的左侧人物照片展示，改为完整显示上传图或 CampusVision C1 代表人脸，避免界面裁掉主要人脸区域。
- 检索记录列表现在会优先显示 CampusVision C1 返回的关键帧缩略图，只有在缺少 `frameUrl` 时才回退默认人物占位图。
- 详情区关键帧、记录缩略图和 CampusVision C1 媒体代理字段保持一致，便于后续接入真实媒体服务。
- 更新静态资源缓存标识，确保安装包和浏览器加载本次 UI 修复。
- README、API 合同、CampusVision C1 / GKGuard C2 集成说明、数据字典、演示脚本和安全支持表已同步到 `v0.1.11` 当前状态。

## 运行说明

- 下载 `GKGuard-Setup-0.1.11.exe` 安装后，打开即可启动内置本地 GKGuard C2 后端并进入 `/demo`。
- CampusVision C1 自动连接、内置服务器候选地址、SSH 密码提示、mock fallback、Electron 42 和 Node.js 22 workflow 保持不变。
- C1 已连接时，结果页应显示 `C1 CampusVision` 数据来源、真实关键帧缩略图和详情关键帧。
- CampusVision C1 不可用或未上传图片时，仍可用本地模拟数据完成三屏演示。

## 验证情况

- 后端与静态资源回归测试：`22 passed, 1 warning`。
- Electron 主进程语法检查：`node --check desktop/main.js` 通过。
- `git diff --check` 通过。
- 本地浏览器试用通过：结果页人物照片为 `object-fit: contain`，5 条 CampusVision C1 样例记录均显示关键帧缩略图，详情关键帧正常显示。
- `npm ci` 通过，`npm audit --audit-level=low` 确认 0 vulnerabilities。
- PyInstaller 后端包构建通过，本地 Windows 安装包构建 `npm run dist` 通过。

## 安装包

本 Release 下方包含：

- `GKGuard-Setup-0.1.11.exe`：Windows 桌面安装包。
- `GKGuard-Setup-0.1.11.exe.blockmap`：Electron 更新差分元数据。
- `latest.yml`：Electron 更新通道元数据。

普通安装只需要下载并运行 `GKGuard-Setup-0.1.11.exe`。

# GKGuard v0.1.11

## English

## Highlights

- Improved the target portrait on the face-search result screen so the uploaded image or CampusVision C1 representative face is shown without cutting off the main face area.
- The search record list now prefers CampusVision C1 keyframe thumbnails and falls back to the default person placeholder only when `frameUrl` is missing.
- The detail keyframe, record thumbnails, and CampusVision C1 media proxy fields now use the same normalized media path for easier real-service integration.
- Updated the static asset cache key so the installer and browser load this UI fix.
- Updated README, API contract, CampusVision C1 / GKGuard C2 integration notes, data dictionary, demo script, and security support table for the current `v0.1.11` state.

## Runtime Notes

- Download and install `GKGuard-Setup-0.1.11.exe`, then open the app to start the bundled local GKGuard C2 backend and load `/demo`.
- CampusVision C1 auto-connection, built-in server candidates, SSH password prompt, mock fallback, Electron 42, and the Node.js 22 workflow remain unchanged.
- With C1 connected, the result screen should show `C1 CampusVision`, real keyframe thumbnails, and the selected detail keyframe.
- If CampusVision C1 is unavailable or no image is uploaded, the local mock fallback still supports the three-screen demo.

## Verification

- Backend and static asset regression tests: `22 passed, 1 warning`.
- Electron main-process syntax: `node --check desktop/main.js` passed.
- `git diff --check` passed.
- Local browser trial passed: the target portrait uses `object-fit: contain`, all 5 sample CampusVision C1 records display keyframe thumbnails, and the detail keyframe is visible.
- `npm ci` passed, and `npm audit --audit-level=low` confirmed 0 vulnerabilities.
- PyInstaller backend bundling passed, and the local Windows installer build `npm run dist` passed.

## Installer Assets

This Release includes:

- `GKGuard-Setup-0.1.11.exe`: Windows desktop installer.
- `GKGuard-Setup-0.1.11.exe.blockmap`: Electron differential update metadata.
- `latest.yml`: Electron update-channel metadata.

For normal installation, download and run `GKGuard-Setup-0.1.11.exe`.
