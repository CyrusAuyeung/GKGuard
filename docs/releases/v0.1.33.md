# GKGuard v0.1.33

## 重点更新

- 修复结果页左侧人物照片在单人图、低置信候选和靠近图像边缘的人脸场景中仍可能显示不全的问题。
- 结果页人物照片现在从用户选中的目标框生成扩边裁切图，空间允许时调整为方形裁切，并在裁切框靠近图像边缘时向内平移，尽量保留完整人脸。
- 人物照片方框内的图片元素固定在 8px 安全边距内，并使用 `object-fit: contain` 充分利用方框空间，避免高置信多人场景中裁切图过小。
- 补充 E2E 回归：覆盖单人低置信查询图、多人选中查询图、裁切图扩边、接近方形裁切、人物照片方框空间利用和无页面级横向溢出。
- 同步 README、API 规范、CampusVision C1 / GKGuard C2 集成说明、演示脚本、文档索引、图标说明和版本号到 `v0.1.33`。

## 行为说明

- 本版只调整 GKGuard C2 前端查询人脸裁切和结果页人物照片展示逻辑；不修改远端 CampusVision C1 服务。
- 检索相似度、排序、阈值和 CampusVision C1 返回的关键帧/人脸缩略图逻辑保持不变。
- 本版本不向仓库加入真实监控、人脸或个人信息数据；本地测试图片继续位于被 git 忽略的 `local-test-data/`。

# GKGuard v0.1.33

## Highlights

- Fixed result-side portraits that could still show incomplete faces for single-face uploads, low-confidence candidates, and faces near image edges.
- Result portraits now use a padded crop generated from the user-selected target box, make the crop square when source-image space allows, and shift the crop inward near image edges to preserve the full face where possible.
- The portrait-frame image element is fixed inside an 8px safe margin and uses `object-fit: contain` so the available frame space is used consistently, avoiding undersized crops in high-confidence multi-face scenes.
- Added E2E regression coverage for single low-confidence query images, multi-face selected query images, padded portrait crops, near-square crops, portrait-frame space usage, and no page-level horizontal overflow.
- Synchronized README, API specification, CampusVision C1 / GKGuard C2 integration notes, demo script, documentation index, icon notes, and package versions to `v0.1.33`.

## Behavior Notes

- This release only adjusts GKGuard C2 frontend query-face cropping and result-portrait rendering; it does not modify the remote CampusVision C1 service.
- Similarity scoring, ranking, thresholds, and CampusVision C1 keyframe/face-thumbnail responses remain unchanged.
- This release does not add real surveillance, face, or personal-information data to the repository; local test images remain under git-ignored `local-test-data/`.
