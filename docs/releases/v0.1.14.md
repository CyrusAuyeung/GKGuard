# GKGuard v0.1.14

## 更新重点

- 桌面端更新入口改为内嵌更新流程：检查到新版后在应用内下载，下载完成后点击 `重启安装` 完成升级。
- C1 服务器连接改为软件内 SSH 密码窗口，不再打开外部 PowerShell；密码只用于本次 SSH 隧道连接，不保存、不落盘、不写日志。
- 结果页和路线页新增 `重新上传` 入口，搜索完成后可直接返回上传页选择新的目标照片。
- 桌面端继续优先使用 `http://127.0.0.1:18000` 隧道候选；C1 502/503/504 时仍会尝试下一个候选地址并触发连接重试。
- README、API 合同、C1/C2 集成说明、自动连接说明、演示脚本、安全支持范围、Issue 模板和 Release notes 已同步到 `v0.1.14` 当前状态。

## 使用说明

- 从 `v0.1.13` 升级到本版时，旧版仍会通过现有入口下载 `GKGuard-Setup-0.1.14.exe`；安装本版后，后续更新可在应用内下载并重启安装。
- 打开软件后如 C1 隧道未连接，会显示内嵌 `连接 C1 服务器` 密码窗口。输入服务器密码后软件会建立本机 `18000 -> 127.0.0.1:8000` SSH 隧道。
- 检索完成后，点击结果页或路线页的 `重新上传` 可清空当前目标照片并回到上传页。

## 验证

- 通过：`python -m pytest backend`，24 passed，1 个上游 deprecation warning。
- 通过：`node --check desktop/main.js` 和 `node --check desktop/preload.js`。
- 通过：`npm ci` 和 `npm audit --audit-level=low`，0 vulnerabilities。
- 通过：`npm run dist`，已生成 `GKGuard-Setup-0.1.14.exe` 和 `.blockmap`。

## Release 资产

- `GKGuard-Setup-0.1.14.exe`：Windows 桌面安装包。
- `GKGuard-Setup-0.1.14.exe.blockmap`：Electron 更新差分元数据。
- `latest.yml`：Electron 自动更新元数据。

普通首次安装仍下载并运行 `GKGuard-Setup-0.1.14.exe`。

# GKGuard v0.1.14

## Highlights

- The desktop update entry now uses an embedded update flow: after a newer version is found, the app downloads it in-app and finishes the upgrade through `重启安装`.
- C1 server connection now uses an embedded SSH password prompt instead of an external PowerShell window. The password is used only for the current SSH tunnel and is not stored, written to disk, or logged.
- The result and route screens now include `重新上传`, letting users return to the upload screen and choose a new target after a search.
- The desktop app continues to prefer the `http://127.0.0.1:18000` tunnel candidate; C1 502/503/504 responses still trigger candidate retry and connection retry.
- README, API contract, C1/C2 integration notes, auto-connection guide, demo script, supported security version, issue templates, and Release notes are synced to the current `v0.1.14` state.

## Usage Notes

- When upgrading from `v0.1.13`, the old updater entry still downloads `GKGuard-Setup-0.1.14.exe`; after this version is installed, future updates can download in-app and restart to install.
- If the C1 tunnel is not connected after opening the app, the embedded `连接 C1 服务器` password window appears. After the server password is entered, the app creates the local `18000 -> 127.0.0.1:8000` SSH tunnel.
- After a search finishes, click `重新上传` on the result or route screen to clear the current target image and return to the upload screen.

## Verification

- Passed: `python -m pytest backend`, 24 passed with 1 upstream deprecation warning.
- Passed: `node --check desktop/main.js` and `node --check desktop/preload.js`.
- Passed: `npm ci` and `npm audit --audit-level=low`, 0 vulnerabilities.
- Passed: `npm run dist`, producing `GKGuard-Setup-0.1.14.exe` and `.blockmap`.

## Release Assets

- `GKGuard-Setup-0.1.14.exe`: Windows desktop installer.
- `GKGuard-Setup-0.1.14.exe.blockmap`: Electron differential update metadata.
- `latest.yml`: Electron auto-update metadata.

For a first-time installation, download and run `GKGuard-Setup-0.1.14.exe`.
