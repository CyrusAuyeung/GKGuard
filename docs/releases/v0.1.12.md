# GKGuard v0.1.12

## 中文

## 重点更新

- 修复上传页和结果页人物照片仍可能显示紧裁剪 CampusVision C1 人脸的问题：界面现在优先保留用户上传原图，只有没有上传图时才使用 CampusVision C1 代表人脸。
- 人物照片展示框调整为更适合正脸图片的方形比例，并保持 `object-fit: contain` 与居中展示，避免额头、下巴或主体人脸被裁掉。
- 检索记录缩略图兼容 `thumbnailUrl`、`frameUrl` 和 `faceUrl`，并在图片加载失败时自动回退占位图。
- 桌面模式新增右上角 `检查更新` 入口：软件内可查询 GitHub 最新 Release，发现新版后直接下载最新 `GKGuard-Setup-*.exe`。
- README、API 合同、CampusVision C1 / GKGuard C2 集成说明、演示脚本、安全支持范围和 Release notes 已同步到 `v0.1.12` 当前状态。

## 运行说明

- 下载 `GKGuard-Setup-0.1.12.exe` 安装后，打开即可启动内置本地 GKGuard C2 后端并进入 `/demo`。
- 后续更新无需手动到 GitHub 页面查找安装包；点击软件右上角 `检查更新`，发现新版后再次点击即可下载。
- CampusVision C1 自动连接、内置服务器候选地址、SSH 密码提示、mock fallback、Electron 42 和 Node.js 22 workflow 保持不变。

## 验证情况

- 后端、静态资源和桌面 bridge 回归测试：`23 passed, 1 warning`。
- Electron 主进程语法检查通过。
- 本地浏览器试用通过：上传页头像为 176x176 方形 `contain` 居中，CampusVision C1 返回紧裁剪图时结果页仍显示上传原图，5 条样例记录显示缩略图。
- `npm ci` 通过，`npm audit --audit-level=low` 确认 0 vulnerabilities。
- PyInstaller 后端包构建通过，本地 Windows 安装包构建 `npm run dist` 通过。

## 安装包

本 Release 下方包含：

- `GKGuard-Setup-0.1.12.exe`：Windows 桌面安装包。
- `GKGuard-Setup-0.1.12.exe.blockmap`：Electron 更新差分元数据。
- `latest.yml`：Electron 更新通道元数据。

普通安装只需要下载并运行 `GKGuard-Setup-0.1.12.exe`。

# GKGuard v0.1.12

## English

## Highlights

- Fixed the remaining portrait-cropping issue on the upload and result screens: the UI now preserves the full user-uploaded image first and only uses the CampusVision C1 representative face when no upload is available.
- Changed portrait frames to a face-friendly square ratio while keeping `object-fit: contain` and centered positioning, preventing the forehead, chin, or main face area from being cut off.
- Search record thumbnails now accept `thumbnailUrl`, `frameUrl`, or `faceUrl`, and fall back to the placeholder only when the image fails to load.
- Added a top-right `检查更新` entry in desktop mode: the app can check the latest GitHub Release and directly download the newest `GKGuard-Setup-*.exe`.
- Updated README, API contract, CampusVision C1 / GKGuard C2 integration notes, demo script, supported security version, and Release notes for the current `v0.1.12` state.

## Runtime Notes

- Download and install `GKGuard-Setup-0.1.12.exe`, then open the app to start the bundled local GKGuard C2 backend and load `/demo`.
- Future upgrades no longer require manually finding the installer on GitHub; click the top-right `检查更新`, then click again to download when a newer version is found.
- CampusVision C1 auto-connection, built-in server candidates, SSH password prompt, mock fallback, Electron 42, and the Node.js 22 workflow remain unchanged.

## Verification

- Backend, static asset, and desktop bridge regression tests: `23 passed, 1 warning`.
- Electron main-process syntax check passed.
- Local browser trial passed: the upload portrait is a 176x176 square with centered `contain`, the result screen still shows the uploaded full image when CampusVision C1 returns a tight crop, and all 5 sample records show thumbnails.
- `npm ci` passed, and `npm audit --audit-level=low` confirmed 0 vulnerabilities.
- PyInstaller backend bundling passed, and the local Windows installer build `npm run dist` passed.

## Installer Assets

This Release includes:

- `GKGuard-Setup-0.1.12.exe`: Windows desktop installer.
- `GKGuard-Setup-0.1.12.exe.blockmap`: Electron differential update metadata.
- `latest.yml`: Electron update-channel metadata.

For normal installation, download and run `GKGuard-Setup-0.1.12.exe`.
