# GKGuard v0.1.8

## 中文

## 重点更新

- GKGuard C2 已接入真实 CampusVision C1 服务，不再只是前端 mock 展示。
- 新增 GKGuard C2 后端 `/c1/...` 代理层，用于转发 CampusVision C1 健康检查、人物库、视频列表、以图搜人和媒体帧访问。
- 人脸检索页上传图片后，会通过 GKGuard C2 调用 CampusVision C1 的 `person-by-image` 检索接口，并在结果页展示真实关键帧、相似度、摄像头和时间信息。
- 路线图页面已支持根据 CampusVision C1 返回的轨迹点动态计算出现次数、起止时间、持续时间、摄像头数量、关键帧数量和最终相似度。
- 保留 mock fallback：当未上传图片或 CampusVision C1 暂不可用时，演示流程仍可离线展示完整三屏壳体。
- 仓库纳入 `services/campusvision-c1/` 作为 CampusVision C1 服务源码与实现参考，便于 GKGuard C2 后续真实部署。

## 接口与集成

- `GET /c1/status`：检查 CampusVision C1 可用性和识别引擎。
- `GET /c1/persons`：获取 CampusVision C1 人物库。
- `GET /c1/videos`：获取 CampusVision C1 已上传/已索引视频。
- `POST /c1/search/person-by-image`：GKGuard C2 统一入口，前端字段名为 `file`，由 GKGuard C2 转发给 CampusVision C1 的 `files`。
- `GET /c1/media/frame/{face_id}`：通过 GKGuard C2 代理访问 CampusVision C1 关键帧。
- `GET /c1/media/face/{face_id}`：通过 GKGuard C2 代理访问 CampusVision C1 人脸裁剪图。

## 运行说明

- 下载 `GKGuard-Setup-0.1.8.exe` 安装后，直接打开即可启动内置本地 GKGuard C2 后端并进入 `/demo`。
- 软件本体和 mock fallback 不依赖校园网；不在校园网时也能演示三屏壳体、路线图和本地模拟数据。
- 真实 CampusVision C1 检索需要能访问 CampusVision C1 服务所在服务器，通常需要校园网、VPN 或 SSH 隧道。
- GKGuard C2 默认连接 `C1_BASE_URL=http://127.0.0.1:18000`。
- 如果 CampusVision C1 在服务器上运行，可使用 SSH 隧道映射到本机 `127.0.0.1:18000`。
- CampusVision C1 服务应使用 `FACE_ENGINE=insightface`，否则视频索引、人物库和查询图片可能不在同一向量空间。
- CampusVision C1 运行数据、视频、抽帧、人脸图片、SQLite 数据库和 `.env` 不应提交到仓库。

## 文档更新

- 更新主 README，说明当前 GKGuard C2、CampusVision C1 和 CampusCar/UE 的边界。
- 更新 CampusVision C1 / GKGuard C2 集成文档，明确 GKGuard C2 代理、CampusVision C1 远端服务和 fallback 行为。

# GKGuard v0.1.8

## English

## Highlights

- GKGuard C2 now connects to the real CampusVision C1 service instead of relying only on frontend mock data.
- Added a GKGuard C2 backend `/c1/...` proxy layer for CampusVision C1 health checks, person index, video list, image-based person search, and media-frame access.
- After a face image is uploaded on the search screen, the frontend calls CampusVision C1 through GKGuard C2 and renders real keyframes, similarity scores, cameras, and time information on the result screen.
- The route screen now computes appearance count, start/end time, duration, camera count, keyframe count, and final similarity from CampusVision C1 route points.
- Mock fallback remains available: if no image is uploaded or CampusVision C1 is unavailable, the three-screen shell still works offline for demos.
- `services/campusvision-c1/` is now included as the CampusVision C1 service source and implementation reference for later real deployment.

## APIs And Integration

- `GET /c1/status`: checks CampusVision C1 availability and recognition engine.
- `GET /c1/persons`: returns the CampusVision C1 person index.
- `GET /c1/videos`: returns uploaded or indexed CampusVision C1 videos.
- `POST /c1/search/person-by-image`: unified GKGuard C2 entrypoint; the frontend sends field `file`, and GKGuard C2 forwards it to CampusVision C1 as `files`.
- `GET /c1/media/frame/{face_id}`: proxies CampusVision C1 keyframe media through GKGuard C2.
- `GET /c1/media/face/{face_id}`: proxies CampusVision C1 face-crop media through GKGuard C2.

## Runtime Notes

- After installing `GKGuard-Setup-0.1.8.exe`, opening the app starts the bundled local GKGuard C2 backend and loads `/demo`.
- The app itself and mock fallback do not require campus network access; outside campus network it can still demonstrate the three-screen shell, route map, and local mock data.
- Real CampusVision C1 search requires access to the CampusVision C1 server, usually through campus network, VPN, or an SSH tunnel.
- GKGuard C2 defaults to `C1_BASE_URL=http://127.0.0.1:18000`.
- If CampusVision C1 runs on a remote server, use an SSH tunnel to expose it locally at `127.0.0.1:18000`.
- CampusVision C1 should run with `FACE_ENGINE=insightface`; otherwise video indexing, person indexing, and query images may not share the same embedding space.
- CampusVision C1 runtime data, videos, extracted frames, face images, SQLite databases, and `.env` files should not be committed.

## Documentation Updates

- Updated the main README to reflect current GKGuard C2, CampusVision C1, and CampusCar/UE boundaries.
- Updated the CampusVision C1 / GKGuard C2 integration document to describe the GKGuard C2 proxy, remote CampusVision C1 service, and fallback behavior.
- Updated API contract, data dictionary, demo script, and CampusCar/UE integration contract.
- Converted repository Markdown documentation to Chinese-first bilingual format with an English jump link.

## Verification

- Verified CampusVision C1 `/health` returns `face_engine=insightface`.
- Verified native CampusVision C1 person search returns 200.
- Verified GKGuard C2 proxy person search returns CampusVision C1 records.
- Verified the frontend result screen shows `C1 CampusVision` and a real keyframe.
