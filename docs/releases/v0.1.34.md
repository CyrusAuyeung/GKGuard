# GKGuard v0.1.34

## 重点更新

- 桌面端发布从 Windows 扩展到 Windows、macOS 和 Linux 三个平台。
- Release workflow 改为多平台矩阵构建：Windows 生成 `GKGuard-Setup-*.exe`，macOS 生成 `GKGuard-macOS-*.dmg` / `GKGuard-macOS-*.zip`，Linux 生成 `GKGuard-Linux-*.AppImage` / `GKGuard-Linux-*.deb`，并统一发布到同一个 GitHub Release。
- Electron 主进程按当前系统选择内置 GKGuard C2 后端可执行文件：Windows 使用 `gkguard-backend.exe`，macOS/Linux 使用 `gkguard-backend`。
- 检查更新逻辑改为按平台选择 Release 资产；Windows 继续保留旧版客户端依赖的 `GKGuard-Setup-*.exe` 命名，确保现有 Windows 安装版仍可从软件内更新。
- 新增 `AGENTS.md`，集中记录 AI agent 开发、验证、文档同步、Project 和发布规范，并在 README、贡献指南、文档索引和 PR 模板中加入入口。
- 新增 macOS `app-mark.icns`，桌面端图标资产覆盖 Windows ICO、macOS ICNS 和通用 PNG。

## 行为说明

- Windows 版应用内更新路径保持兼容；已安装旧版 Windows 软件仍可通过右上角 `检查更新` 下载新版 Windows 安装包。
- macOS/Linux 版当前按平台打开 GitHub Release 中对应安装文件；macOS 包暂未配置 Apple 签名与公证。
- 本版只修改 GKGuard C2 桌面端打包、更新资产选择、Release workflow 和协作文档；不修改远端 CampusVision C1 服务、CampusVision C1 检索算法或真实数据索引。
- 本版本不向仓库加入真实监控、人脸或个人信息数据。

## 验证

- 通过：`node --check backend/app/static/app.js`。
- 通过：`node --check desktop/main.js`。
- 通过：`node --check desktop/preload.js`。
- 通过：`python -m py_compile backend/desktop_server.py`。
- 通过：`python -m pytest backend`。
- 通过：`npm run test:e2e`。
- 通过：`npm audit --audit-level=low`。
- 通过：`git diff --check`。
- 通过：`npm run dist:win`，本地生成 Windows 安装包、`.blockmap` 和 `latest.yml`。
- macOS 和 Linux 安装文件以 GitHub Actions `release-desktop.yml` 对应 runner 的构建结果为准。
- 发布流程要求 GitHub Actions `Verify` 和 Release workflow 通过，并在正式 GitHub Release 中核对 Windows、macOS、Linux 资产和 Electron 更新元数据齐全。

## 安装

- Windows：下载并运行 `GKGuard-Setup-0.1.34.exe`。已安装旧版时，可使用软件右上角 `检查更新` 入口更新。
- macOS：下载 `GKGuard-macOS-*.dmg` 或 `GKGuard-macOS-*.zip` 试用桌面版；当前包未签名、未公证，可能需要在系统安全设置中手动允许打开。
- Linux：下载 `GKGuard-Linux-*.AppImage` 或 `GKGuard-Linux-*.deb` 试用桌面版。

真实 CampusVision C1 检索仍需要校园网、VPN 或 SSH 隧道能访问 CampusVision C1 服务；未上传图片时可使用本地模拟流程演示，上传图片后的检测失败、无匹配、超时或真实检索失败不会展示模拟命中。

# GKGuard v0.1.34

## Highlights

- Extended desktop releases from Windows-only to Windows, macOS, and Linux.
- Converted the Release workflow to a multi-platform build matrix: Windows produces `GKGuard-Setup-*.exe`, macOS produces `GKGuard-macOS-*.dmg` / `GKGuard-macOS-*.zip`, Linux produces `GKGuard-Linux-*.AppImage` / `GKGuard-Linux-*.deb`, and one publish job attaches all assets to the same GitHub Release.
- The Electron main process now selects the bundled GKGuard C2 backend executable by platform: `gkguard-backend.exe` on Windows and `gkguard-backend` on macOS/Linux.
- Update checking now selects Release assets by platform. Windows keeps the `GKGuard-Setup-*.exe` naming required by existing Windows clients so installed Windows builds can continue updating from inside the app.
- Added `AGENTS.md` as the central AI agent guide for development, validation, documentation synchronization, Project tracking, and releases, with references from README, the contributing guide, the docs index, and the PR template.
- Added macOS `app-mark.icns`; desktop icon assets now cover Windows ICO, macOS ICNS, and shared PNG.

## Behavior Notes

- The Windows in-app update path remains compatible. Existing Windows installations can still download the new Windows installer through the top-right `检查更新` entry.
- The macOS/Linux app currently opens the matching platform asset on GitHub Releases. Apple signing and notarization are not configured yet for the macOS package.
- This release only changes GKGuard C2 desktop packaging, update asset selection, the Release workflow, and collaboration documentation. It does not modify the remote CampusVision C1 service, CampusVision C1 search algorithms, or real data indexes.
- This release does not add real surveillance, face, or personal-information data to the repository.

## Verification

- Passed: `node --check backend/app/static/app.js`.
- Passed: `node --check desktop/main.js`.
- Passed: `node --check desktop/preload.js`.
- Passed: `python -m py_compile backend/desktop_server.py`.
- Passed: `python -m pytest backend`.
- Passed: `npm run test:e2e`.
- Passed: `npm audit --audit-level=low`.
- Passed: `git diff --check`.
- Passed: `npm run dist:win`, generating the Windows installer, `.blockmap`, and `latest.yml` locally.
- macOS and Linux packages are verified by the corresponding GitHub Actions runners in `release-desktop.yml`.
- Release flow requires GitHub Actions `Verify` and the Release workflow to pass, followed by checking that the official GitHub Release contains complete Windows, macOS, Linux, and Electron update assets.

## Installation

- Windows: download and run `GKGuard-Setup-0.1.34.exe`. Existing installations can update through the top-right `检查更新` entry.
- macOS: download `GKGuard-macOS-*.dmg` or `GKGuard-macOS-*.zip` to try the desktop app. The current package is unsigned and unnotarized, so macOS may require manually allowing it in system security settings.
- Linux: download `GKGuard-Linux-*.AppImage` or `GKGuard-Linux-*.deb` to try the desktop app.

Real CampusVision C1 search still requires campus network, VPN, or an SSH tunnel to the CampusVision C1 service. The mock flow remains available when no image has been uploaded; uploaded-image detection failures, no-match results, timeouts, or real-search failures do not show mock hits.
